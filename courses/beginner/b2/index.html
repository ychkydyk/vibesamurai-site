B2 · Установка вашего первого AI-агента

Ставим Claude Code на Windows/Mac/Linux. Регистрация, ключ, первая команда — и кратко об альтернативах.

Черновик B2 в голосе DmitriyM (см. scripts/tg_voice_analysis/_summary.md).

Hook

ВЫ           +   ТЕРМИНАЛ   +   claude    =    первый рабочий агент
(человек)        (окошко)       (команда)

Друзья, всем привет! 👋

В этой главе мы ставим Claude Code — то самое волшебное claude в командной строке, которое потом будет делать всю основную работу в нашем курсе. Двадцать минут, и у вас на компьютере появляется AI-агент. Это совсем не страшно — обещаю.

Концепция: что такое CLI и почему мы выбрали именно Claude Code

CLI (Command Line Interface) — это просто окошко терминала, в которое вы пишете команды и получаете ответы. Не графический интерфейс с кнопками — текстовый.

Звучит архаично, но в 2026-м это самый удобный формат работы с AI-агентом. Почему:

  • Агенту удобно «видеть» вашу папку и работать с файлами
  • Вам удобно копировать команды из учебных материалов
  • Между разными ОС всё работает одинаково

Среди тулов мы начнём с Claude Code по трём причинам:

  1. Самая дружелюбная установка для новичка
  2. Хорошие defaults — не надо сразу настраивать 30 параметров
  3. Подробные подсказки прямо в чате — задаёте вопрос, агент объясняет, как им пользоваться

В B8 мы посмотрим на альтернативы (Cursor, Codex CLI). Сейчас — Claude Code.

Пример: ставим за 5 минут

Шаг 1. Открыть терминал

Ваша ОСЧто нажать
macOSSpotlight (⌘ + пробел) → «Terminal» → Enter
Windows 11Start → «Terminal» (это новый Windows Terminal)
LinuxCtrl + Alt + T

Откроется чёрное (или белое) окошко с курсором. Это и есть терминал. Не пугайтесь — мы туда введём пару команд.

Шаг 2. Поставить Node.js (если ещё нет)

Claude Code работает поверх Node.js — стандартного движка для запуска современных CLI-инструментов. Проверьте, есть ли он:

node --version

Если выводит что-то вроде v22.x.x — отлично, идём дальше. Если ошибка «command not found» — ставим:

  • macOS: brew install node (если есть Homebrew) или скачать с nodejs.org
  • Windows: скачать установщик с nodejs.org, запустить, всё «Next, Next, Finish»
  • Linux: sudo apt install nodejs npm (Ubuntu/Debian) или аналог для вашего дистрибутива

После установки закройте и снова откройте терминал, иначе node --version всё равно не увидит.

Шаг 3. Поставить Claude Code

npm install -g @anthropic-ai/claude-code

npm — это менеджер пакетов Node.js. Команда говорит «установи глобально (-g) пакет @anthropic-ai/claude-code». Пройдёт пара минут.

Проверка:

claude --version

Если выводит версию — установка прошла. Полный фарш, поздравляю.

Шаг 4. Залогиниться

Просто наберите в терминале:

claude

Откроется браузер с предложением залогиниться в Anthropic-аккаунт. Если у вас уже есть подписка Claude Pro/Max — отлично, входите через неё. Если нет — можно начать с pay-per-use API, для обучения хватит 5-10 долларов на первые недели.

Лично у меня — Max-подписка ($100/мес). Для ежедневной работы окупается в первую неделю, но для обучения переплата. Начните с минимума.

Шаг 5. Первая команда

После логина вы окажетесь в интерактивном режиме Claude Code. Создайте пустую папку для экспериментов:

mkdir my-first-agent
cd my-first-agent
claude

И в открывшемся чате напишите:

«Привет! Расскажи коротко о себе и что ты умеешь.»

Агент представится, расскажет о тулах, объяснит правила. Поздравляю — первый разговор с вашим AI-агентом состоялся.

Practice (20-30 минут)

Задание 1 (10 минут) — установка

Пройдите все 5 шагов выше до конца. Если на каком-то споткнулись — гугл / задайте вопрос в комментариях курса. Не пропускайте. В следующей главе мы уже работаем в Claude Code на полную.

Задание 2 (15 минут) — первый «безопасный» запрос

В новой сессии (claude в любой папке) попросите:

«Прочитай все файлы в этой папке и кратко опиши, что тут лежит. Ничего не меняй.»

Если папка пустая — он скажет «пусто». Если в ней что-то есть — опишет. Это режим «только посмотреть» из B1. Закрепите привычку открывать сессию с него.

Recap

  • Claude Code = CLI-агент от Anthropic. Ставится через npm.
  • Pre-requirement: Node.js. Поставили его — ставите claude.
  • Первая сессия — claude в нужной папке + «только посмотреть».

Вайб Самурай

🥷 Вайб Самурай спрашивает: «А зачем мне всё это, если есть ChatGPT в браузере? Там же тоже можно код просить.»

Ответ: ChatGPT в браузере вам показывает текст. Дальше вы сами копируете его в файлы, проверяете, запускаете. Каждый раз. Claude Code сам пишет в ваши файлы, запускает команды и проверяет результат. Разница не в умности модели — она и там, и там Claude. Разница в руках: у Claude Code они есть. Через две недели работы возвращаться к копированию из браузера будет физически больно.

Дальше

В B3 — Первый разговор разбираем, как просить агента так, чтобы он понял вас с первого раза. Структура запроса, типичные ошибки, когда соглашаться, а когда «нет, переделай».

Если зашло — поделитесь главой с тем, кто давно хочет «потрогать AI», но не знает с чего начать. Если что-то на установке не пошло — пишите в комменты, обычно это решается за две минуты, ваша обратная связь реально помогает шлифовать инструкции.

📚 Дальше: посмотреть и почитать

Если на установке где-то застряли или хочется сразу глянуть, как настраивается Claude Code «по-взрослому» — вот короткий список первоисточников, на которые я сам сошлюсь во всех следующих главах.

Официальные доки

  • Claude Code — Advanced setup (официальная инструкция Anthropic, ставит npm/native/Homebrew): code.claude.com/docs/en/setup. Если что-то не пошло на шаге 3 — открывайте сюда первой.
  • Claude API — Getting started (как устроены API-ключи, если решили идти через pay-per-use, а не подписку): docs.anthropic.com/en/api/getting-started.
  • Anthropic Console — API keys (страница, где, собственно, создаётся ключ): console.anthropic.com/settings/keys. Ключ показывается один раз — скопируйте сразу.

GitHub-репозитории

  • anthropics/claude-code — официальный репозиторий, оттуда видны актуальные релизы и issues: github.com/anthropics/claude-code.

Совет: прежде чем спрашивать «почему не работает» — пробегитесь по странице Advanced setup. 80% типовых ошибок установки решаются её первыми двумя разделами.
