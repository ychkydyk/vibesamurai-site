B2 · Installing your first AI agent

Putting Claude Code on Windows/Mac/Linux. Sign up, get a key, run the first command — and a short word on alternatives.

Hook

YOU          +   TERMINAL   +   claude    =    first working agent
(human)          (window)       (command)

Hey folks, welcome 👋

In this chapter we install Claude Code — the magic claude command in the terminal that will do most of the heavy lifting for the rest of the course. Twenty minutes and you have an AI agent on your machine. Not scary at all — promise.

Concept: what’s a CLI and why we start with Claude Code

CLI (Command Line Interface) is just a terminal window where you type commands and get answers. Not a graphical interface with buttons — text.

Sounds archaic, but in 2026 it’s the most convenient format to work with an AI agent. Why:

  • The agent finds it easy to “see” your folder and work with files
  • You find it easy to copy commands from training materials
  • It works the same across operating systems

Among the tools we start with Claude Code for three reasons:

  1. Friendliest install for a newcomer
  2. Good defaults — you don’t have to configure 30 parameters first
  3. Detailed prompts right inside the chat — ask a question, the agent explains how to use it

In B8 we look at alternatives (Cursor, Codex CLI). For now — Claude Code.

Example: install in 5 minutes

Step 1. Open a terminal

Your OSWhat to press
macOSSpotlight (⌘ + Space) → “Terminal” → Enter
Windows 11Start → “Terminal” (the new Windows Terminal)
LinuxCtrl + Alt + T

A black (or white) window with a cursor opens. That’s the terminal. Don’t panic — we’ll type a few commands.

Step 2. Install Node.js (if you don’t have it)

Claude Code runs on top of Node.js — the standard engine for running modern CLI tools. Check:

node --version

If you see something like v22.x.x — great, move on. If it says “command not found” — install:

  • macOS: brew install node (if you have Homebrew) or download from nodejs.org
  • Windows: download the installer from nodejs.org, run, “Next, Next, Finish”
  • Linux: sudo apt install nodejs npm (Ubuntu/Debian) or the equivalent for your distro

After install close and reopen the terminal, otherwise node --version still won’t see it.

Step 3. Install Claude Code

npm install -g @anthropic-ai/claude-code

npm is Node.js’s package manager. The command says “install globally (-g) the package @anthropic-ai/claude-code.” Takes a minute or two.

Check:

claude --version

If it prints a version — you’re in. Full package, congratulations.

Step 4. Log in

Just type in the terminal:

claude

A browser opens asking you to log into your Anthropic account. If you already have a Claude Pro/Max subscription — perfect, sign in with it. If not — you can start on the pay-per-use API; for learning, $5–10 lasts the first few weeks.

Personally — I’m on the Max plan ($100/mo). For daily work it pays for itself in week one, but for learning it’s overpay. Start with the minimum.

Step 5. First command

After login you land in Claude Code’s interactive mode. Make an empty folder for experiments:

mkdir my-first-agent
cd my-first-agent
claude

In the chat that opens, write:

“Hi! Tell me briefly about yourself and what you can do.”

The agent introduces itself, lists its tools, explains the rules. Congratulations — your first conversation with your AI agent has happened.

Practice (20–30 minutes)

Task 1 (10 minutes) — install

Walk through all 5 steps end-to-end. If you got stuck somewhere — Google / ask in the course comments. Don’t skip. In the next chapter we’re already working in Claude Code full-on.

Task 2 (15 minutes) — first “safe” request

In a fresh session (claude in any folder) ask:

“Read every file in this folder and briefly describe what’s here. Don’t change anything.”

If the folder is empty — it’ll say “empty.” If there’s something — it’ll describe it. That’s the “look, don’t touch” mode from B1. Lock in the habit of opening sessions with this.

Recap

  • Claude Code = a CLI agent from Anthropic. Installs via npm.
  • Pre-requirement: Node.js. Install it, then install claude.
  • First session — claude in the target folder + “look, don’t touch.”

Vibe Samurai

🥷 Vibe Samurai asks: “Why do I need any of this if there’s ChatGPT in the browser? I can ask it for code too.”

Answer: ChatGPT in the browser shows text. Then you copy it into files, check it, run it. Every time. Claude Code writes into your files itself, runs commands itself, and checks the result itself. The difference isn’t model smarts — it’s Claude in both. The difference is hands: Claude Code has them. Two weeks in, going back to “copy from browser” feels physically painful.

📚 Further reading

If you got stuck somewhere on install or want to see how Claude Code gets configured “for grown-ups” — a short list of sources I’ll keep referencing.

Official docs

  • Claude Code — Advanced setup (Anthropic’s official guide, covers npm/native/Homebrew): code.claude.com/docs/en/setup. If something didn’t go through at step 3 — open this first.
  • Claude API — Getting started (how API keys work if you went pay-per-use instead of subscription): docs.anthropic.com/en/api/getting-started.
  • Anthropic Console — API keys (the page where you actually create the key): console.anthropic.com/settings/keys. The key is shown once — copy it immediately.

GitHub repos

  • anthropics/claude-code — the official repo; shows current releases and issues: github.com/anthropics/claude-code.

Tip: before asking “why doesn’t it work” — skim the Advanced setup page. 80 % of common install errors are solved by its first two sections.
