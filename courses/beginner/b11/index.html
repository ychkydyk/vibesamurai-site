B11 · Капстоун: соберите свой инструмент

Финальный проект Beginner-трека. От «учебной задачи» к личному инструменту, которым будете пользоваться сами.

Черновик B11 в голосе DmitriyM (см. scripts/tg_voice_analysis/_summary.md).

Hook

B5: todo для всех         B11: что-то, что
                          лично вам нужно

«учебный проект»     →    «я этим пользуюсь
                           каждый день»

Друзья, всем привет! 👋

Это финальная глава Beginner-трека. Никаких новых концепций — мы собираем всё, что было в B1-B10, в один реальный проект. Главная разница с todo из B5: это не учебная задача. Это штука, которая нужна лично вам, которой вы потом будете сами пользоваться.

Это самое важное, что я могу вам передать в этом треке. Потому что сделать ещё один todo — это упражнение. Сделать свой собственный инструмент — это точка, после которой обратной дороги нет: вы поняли, что сами можете делать программы, которые вам нужны.

Полтора-два часа. Берите вкусный чай.

Концепция: «бытовая программа» как объект разработки

Раньше «программа» = «продукт для миллионов пользователей». Сейчас, в эпоху агентов, у каждого появилось личное право на программу:

┌─────────────────────────────────────────────────────┐
│  Раньше:                                            │
│  «Хочу X» → купить готовое или жить без             │
│                                                     │
│  Теперь:                                            │
│  «Хочу X» → описать агенту → получить               │
└─────────────────────────────────────────────────────┘

Это не теория. Это уже реальность для тех, кто прошёл такой трек, как этот.

Как выбрать идею для капстоуна

Правило 1. Это должно решать вашу собственную боль. Не «я подумал, людям такое надо». А «я этим сам буду пользоваться». Чем чаще — тем лучше.

Правило 2. Реалистичный объём — 15-30 минут работы агента, не «месячный проект».

Правило 3. Локально или на статике. Без сложных серверов на старте.

Примеры (для вдохновения)

  • Личный трекер привычек — таблица «дни / привычки», клик = ✓, сохраняется в браузере
  • Конвертер единиц для вашего хобби (например, фотограф — диафрагмы/выдержки/ISO; бариста — рецепты)
  • Случайный выборщик — «куда сегодня сходить поесть» из вашего списка
  • Калькулятор калорий по вашим стандартным блюдам (не для всех, а ваше меню)
  • Чек-лист «перед отлётом» — повторяющийся, с галочками, сбрасывается после поездки
  • Pomodoro-таймер с вашими интервалами (стандартный 25/5 не для всех)
  • Конспект-генератор — вы пишете 5 buzzwords, выходит структурированный мини-конспект
  • Кошелёк трат за месяц — три кнопки «кофе / еда / транспорт», сохраняется, в конце месяца видна сумма
  • «Что я знаю про X» — личный мини-вики по теме, к которой возвращаетесь
  • Расписание лекарств / БАДов — напоминалка с вашими временами

Эти не «гениальные» идеи. Они полезные именно вам. Это и есть точка.

Шаги капстоуна

Сейчас по порядку, без лишнего. Каждый шаг ссылается на главу, где разбирали детали.

Шаг 1. Идея → формула (10 минут)

Возьмите идею, запишите её по формуле из B3:

ЧТО:
ГДЕ:
КАК ПОЙМЁМ ГОТОВЫМ:

Пример (трекер привычек):

  • ЧТО: сделать веб-страницу-трекер привычек
  • ГДЕ: новая папка ~/projects/habits, без сборки, чистый HTML/CSS/JS
  • КАК ПОЙМЁМ ГОТОВЫМ:
    • В шапке текущая дата
    • Список из 5 привычек (Вода, Тренировка, Чтение, Медитация, Без сахара)
    • Под каждым днём недели — чекбоксы по 5 привычек
    • Клик на чекбокс — сохраняется
    • Перезагрузка — состояние не теряется
    • Кнопка «Новая неделя» — сбрасывает чекбоксы, дату сдвигает

Это и есть ваше ТЗ.

Шаг 2. Сетап (5 минут)

mkdir habits
cd habits
git init
echo ".env" > .gitignore
git add .gitignore
git commit -m "init"
claude

См. B4 (git) и B7 (.gitignore сразу).

Шаг 3. План (10 минут)

Скиньте агенту вашу постановку из Шага 1. Скажите:

«Сначала покажи план — какие файлы, какая структура, без кода.»

Прочитайте план. Если согласны — «делай». Если что-то не так — поправьте сейчас.

См. B5.

Шаг 4. Первая итерация (15-25 минут)

Агент пишет файлы. Запустите → проверьте. Используйте чек-лист из B10:

  • Соответствие критериям
  • Edge case’ы
  • Консоль
  • git diff
  • Перезагрузка

После принятия — коммит (см. B4).

Шаг 5. 2-3 правки (20-30 минут)

Что-то наверняка надо поправить. Сформулируйте по формуле, дайте агенту. После каждой принятой правки:

git add .
git commit -m "<коротко что>"

См. B3 (как сформулировать), B10 (чек-лист), B4 (коммит).

Шаг 6. Деплой (15 минут)

Если ваш проект — статика (HTML/CSS/JS, без бэка):

# Создайте репо на GitHub (см. B9)
git remote add origin https://github.com/ВАШЕ-ИМЯ/habits.git
git branch -M main
git push -u origin main

# Включите GitHub Pages в Settings → Pages

Получите URL. Откройте с телефона.

См. B9.

Шаг 7. Используйте (открытое время)

Главное. Сегодня, завтра, послезавтра — попробуйте пользоваться своей же программой. Заведите её в закладки телефона, проставьте чекбоксы, попробуйте «новую неделю».

После недели использования у вас будет 2-3 правки, которые очевидны изнутри:

  • «Шрифт мелкий, не вижу на бегу»
  • «Хочу добавить шестую привычку»
  • «Хочу видеть прошлую неделю»

Каждая правка — это новая итерация: формула → агент → чек-лист → коммит → деплой. Полный фарш вашего собственного жизненного цикла продукта.

Чего НЕ делать на капстоуне

  • Не выбирайте «мега-идею». «Соцсеть для собачников» — не сегодня. Сегодня — одна маленькая полезная штука.
  • Не пытайтесь добавить login, базу, сложный бэкенд. Локалка/статика, всё в браузере.
  • Не зацикливайтесь на дизайне. Дизайн проще всего улучшить потом. Сначала — функция.
  • Не сидите молча 4 часа без git status. После каждой правки — коммит. Из B4.

Пример: трекер привычек, реальная история

Один из ребят из ранней группы курса сделал такой трекер для себя. Через две недели — прислал скрин: 14 дней подряд все 5 чекбоксов отмечены. Сказал, что бесплатные приложения в маркете его «давили рекламой» и он их забрасывал. Своё — без рекламы, под него, по простоте — не забросил.

Это и есть смысл капстоуна. Не «сделать ещё одну поделку», а получить инструмент, который вы сами выбрали, сами сделали, и которым сами пользуетесь.

Practice — это и есть капстоун

Главное задание (1.5-2 часа) — сделать и задеплоить

Один проект. От формулы до URL на GitHub Pages. Все 7 шагов выше.

Дополнительное задание (день-неделя) — пользоваться

Положите ссылку в закладки. Используйте. Сделайте 2-3 правки на основе реального опыта использования. Закоммитьте, передеплойте.

После недели использования у вас:

  • Свой инструмент в проде
  • 5-10 коммитов в истории
  • Работающий цикл «идея → правка → деплой»
  • Реальное понимание, что вы можете делать свои инструменты

Recap

  • Капстоун = свой инструмент. Не учебка, не «для портфолио», а то, чем сами пользуетесь.
  • Все шаги — из предыдущих глав. Формула (B3), git (B4), end-to-end flow (B5), безопасность (B7), деплой (B9), оценка (B10).
  • Главное — пользоваться. Сделать — это половина. Использовать неделю и доработать — вторая половина и главный сдвиг.

Скептик Самурай

🥷 Скептик Самурай спрашивает: «А если моя идея — не оригинальная, такое уже есть в маркете? Зачем делать своё?»

Ответ: Затем, что в маркете не ваше. Маркетовые приложения универсальные, с рекламой, со всеми возможными фичами, которые вам не нужны. Ваше — заточено под вашу конкретную привычку, без лишнего, на вашем языке, на вашем домене. Аналогия: каждое утро вы могли бы пить кофе в кафе, но дома — быстрее, дешевле, по-вашему. То же с инструментами. И ещё — даже если в маркете «есть», то делая своё, вы тренируете мышцу делать. Через месяц-два это становится сверхспособностью — не зависеть от маркета вообще. Не идеально, но реально работает.

Дальше

В B12 — Куда дальше — финальная глава Beginner-трека. Соберём, что вы умеете на этой точке, и расскажу, куда двигаться — какие есть треки, какие следующие навыки, какие комьюнити. Это выход из новичка в уверенного пользователя.

Если зашло — поделитесь главой с тем, кто знает теорию, но ничего своего не сделал. Капстоун — это точка инициации. Если задеплоили — киньте ссылку в комменты, реально хочется посмотреть, что у вас получилось. За лайк и комментарий — отдельная благодарность, ваша обратная связь реально помогает шлифовать материал.
