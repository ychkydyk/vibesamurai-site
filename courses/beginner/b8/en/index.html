B8 · Cursor vs Claude Code vs Codex CLI

Map of the agent zoo. What to pick by task, what I personally use, and why 'the best' tool doesn't exist.

Hey folks, welcome 👋

This chapter is a map of the zoo. I’ll tell you which main agents exist, how they differ, and what I use myself. No holy wars “X beats Y.” In reality each tool has its own zone, and a normal “vibe-coder” in 2026 usually keeps two or three within reach.

Twenty minutes — and you have a full picture to pick the first or second tool for yourself, not “because a blogger said so.”

Concept: three agent forms

All current agents split into three types by interaction form:

┌─────────────────────────────────────────────────────┐
│  1. CLI       — terminal-based (Claude Code, Codex) │
│  2. IDE       — inside the editor (Cursor, Windsurf)│
│  3. Web       — in the browser (claude.ai, ChatGPT) │
└─────────────────────────────────────────────────────┘

CLI is the most “bare” and universal. IDE is the most comfortable for editor-people. Web is the most limited (no hands, see B2).

In this chapter we compare the first two — CLI and IDE. Web agents as a “primary” tool we don’t recommend.

The big three

Claude Code (Anthropic)

What it is: a CLI agent. You run claude in the project folder, talk via the terminal.

Pros:

  • Strongest defaults for a newcomer — built-in safety, clear prompts
  • Permission model — asks before every dangerous action
  • MCP, hooks, subagents, skills — extends to any workflow
  • CLAUDE.md in the project — settings and context auto-loaded
  • Pro/Max subscription — flat monthly cost, no per-token billing

Cons:

  • Limits on Pro/Max — heavy use can hit a cap (5–6 hours of active session)
  • Less “IDE magic” — no inline tab autocomplete like Cursor
  • CLI — some find text-based work unfamiliar

Best for:

  • Beginners — because of trustable defaults
  • People working on several projects a day — fast switching
  • People who value transparency of agent behaviour

Codex CLI (OpenAI)

What it is: OpenAI’s CLI agent. Similar form to Claude Code (terminal, files, MCP), but under the hood GPT models and an OS-level sandbox.

Pros:

  • Built-in OS sandbox (macOS sandbox-exec, Linux seccomp, Windows Job Objects) — the agent can’t leave the folder/network without explicit permission
  • AGENTS.md — equivalent of CLAUDE.md (often symlinked so both agents read one file)
  • GPT models — different “brain,” sometimes stronger on some tasks, weaker on others
  • OpenAI Plus/Pro — separate subscription, doesn’t eat Claude limits

Cons:

  • Separate subscription ($20–200/month at OpenAI) — duplication if you already have Claude
  • Fewer ready integrations (skills, hooks — simpler than Claude Code so far)
  • Docs are patchy — ecosystem younger

Best for:

  • People who hit Claude limits and don’t want week-long downtime
  • People who want a second brain — ask two models the same question and compare
  • People who value OS-level sandbox (production machines)

Cursor (Cursor)

What it is: a full IDE (VS Code fork) with a built-in AI agent. You open the project like in an editor and work inside it.

Pros:

  • Most comfortable UX for people used to VS Code/JetBrains
  • Composer — mode where the agent works across several files at once
  • AI tab completion — finishes code as you type (the “magic” experience)
  • Reject/Accept on every change — convenient visual control
  • .cursor/rules/ — rules for the agent tied to the project

Cons:

  • Subscription ($20+/month separate from Claude/OpenAI)
  • Locked to one app — if Cursor crashes, you’re stopped
  • Easier to confuse a newcomer — many buttons, modes

Best for:

  • People who already work in VS Code and don’t want the terminal
  • People who need to see code in front of them at all times
  • People making big projects with dozens of files

Less famous but worthy

Windsurf (Codeium)

IDE agent, Cursor’s competitor. Strength — Cascade (agent that understands the whole project’s context). Good in big monorepos. UX is a bit “simpler” than Cursor.

Aider (open-source)

CLI, open-source. Plugs into any model (Claude, GPT, local). Minimalist but powerful for those who want full freedom. Not for the absolute beginner.

Cline (open-source, VS Code plugin)

Free plugin that turns VS Code into something Cursor-like. You bring your Claude/OpenAI key. Excellent if you don’t want a separate Cursor subscription.

My personal stack

As of May 2026, I run:

  • Claude Code Max — primary work, 80–90 % of the time
  • Codex CLI Plus — when I hit Claude limits (happens on big projects late in the day) or for a second opinion
  • Cursor — for long refactors where it’s handy to see diffs
  • Claude.ai in the browser — for purely conversational tasks (plans, explanations, ideas)

This is not “the right way” — it’s what’s comfortable for me. Yours may differ.

Principle: don’t build a “zoo” upfront. Start with one. When you hit its limits — you’ll know which second to add. Not perfect, but actually works.

How to pick the first

If beginner and want a fast start → Claude Code. Softest defaults, best onboarding docs, no configuration needed.

If already used to VS Code and don’t want the terminal → Cursor (or Cline as a free alternative).

If you already have OpenAI Pro and don’t want Anthropic → Codex CLI. Works similarly, ecosystem growing.

If budget is zero → Cline + a free API key (e.g. Anthropic API free tier). Not “super smooth,” but a working combo for learning.

How to add a second one (when the time comes)

Signals it’s time to add a second tool:

  1. Hitting limits regularly on one and losing an hour “because the quota ran out”
  2. Noticed a weak spot — e.g. your agent is bad at SQL, the other is great
  3. You want something missing — e.g. OS-sandbox or Composer mode

Then install a second, keep one CLAUDE.md / AGENTS.md (via symlink) per project, and use it per task.

Practice (20 minutes)

Task 1 (10 minutes) — assess your workflow

After a couple of weeks on Claude Code from B2–B7, answer yourself three questions:

  1. Where was it uncomfortable? (Terminal? Limits? Something else?)
  2. What would I like to have? (Inline hints? Sandbox? Second opinion?)
  3. Which tool fills exactly that gap?

If the answers are “all fine” — great, don’t multiply entities. One Claude Code covers most beginners.

Task 2 (10 minutes) — try an alternative

Install one alternative (Cursor or Cline or Codex CLI). Do the same task as in B5 (todo app). Compare:

  • What was smoother?
  • Where did you get confused?
  • In which tool did the code itself come out cleaner?

Note your conclusions — that’s your personal benchmark.

Recap

  • Big three: Claude Code (CLI), Codex CLI (CLI), Cursor (IDE). Plus solid alternatives (Windsurf, Aider, Cline).
  • One tool covers 80 %+ of a beginner’s tasks. Don’t grow a zoo preemptively.
  • Signal to add a second: limits, weak spots, a specific missing feature.

Vibe Samurai

🥷 Vibe Samurai asks: “Why do you recommend Claude Code and not the cheapest/free option? Is this an ad?”

Answer: If Anthropic were free — I’d recommend free. Right now Claude Code is the best onboarding for an absolute beginner among paid, and Cline is the best free. I genuinely work on Claude Max ($100/month) because it pays back in week one — for my work. For your learning — Pro ($20) is plenty, or honestly free Cline on a free-tier API. No ad — I have no relationship with any of these companies. If a tool ships tomorrow that’s clearly better — I’ll recommend it. Not perfect, but actually works — pick by budget and comfort, not by brand.
