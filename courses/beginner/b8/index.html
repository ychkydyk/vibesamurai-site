B8 · Cursor vs Claude Code vs Codex CLI

Карта зоопарка агентов. Что выбрать под задачу, на чём работаю лично я, и почему «лучшего» инструмента нет.

Друзья, всем привет! 👋

Эта глава — карта зоопарка. Я расскажу, какие основные агенты есть, чем они отличаются, и на чём я работаю сам. Без религиозных войн «X лучше Y». В реальности у каждого инструмента есть своя зона, и нормальный «вайб-кодер» в 2026 году обычно держит два-три под рукой.

Двадцать минут — и у вас есть полная картина, чтобы выбрать первый-второй инструмент под себя, а не «потому что блогер сказал».

Концепция: три формы агента

Все современные агенты делятся на три типа по форме взаимодействия:

┌─────────────────────────────────────────────────────┐
│  1. CLI       — терминальный (Claude Code, Codex)   │
│  2. IDE       — внутри редактора (Cursor, Windsurf) │
│  3. Web       — в браузере (claude.ai, ChatGPT)     │
└─────────────────────────────────────────────────────┘

CLI — самый «голый» и универсальный. IDE — самый комфортный для тех, кто привык к редактору. Web — самый ограниченный (нет рук, см. B2).

В этой главе сравниваем первые два — CLI и IDE. Web-агенты как «основной» инструмент мы не рекомендуем.

Большая тройка

Claude Code (Anthropic)

Что это: CLI-агент. Запускаете claude в папке проекта, общаетесь через терминал.

Плюсы:

  • Самые сильные defaults для новичка — встроенная безопасность, чёткие подсказки
  • Permissions модель — спрашивает перед каждым опасным действием
  • MCP, hooks, subagents, skills — расширяется на любой workflow
  • CLAUDE.md в проекте — настройки и контекст автоматически подхватываются
  • Подписка Pro/Max — фикс-цена в месяц, не платите за каждый токен

Минусы:

  • Лимиты на Pro/Max — после интенсивной работы можно упереться (5-6 часов активной сессии)
  • Меньше «магии IDE» — нет автодополнения по ходу, как в Cursor
  • CLI — кому-то непривычно работать через текст

Кому подойдёт:

  • Новичку — потому что доверие к defaults
  • Тем, кто работает с несколькими проектами в день — переключение быстрое
  • Тем, кому важна прозрачность работы агента

Codex CLI (OpenAI)

Что это: CLI-агент от OpenAI. Похож на Claude Code по форме (терминал, файлы, MCP), но под капотом GPT-модели и OS-уровневый sandbox.

Плюсы:

  • Встроенный OS-sandbox (macOS sandbox-exec, Linux seccomp, Windows Job Objects) — агент не может выйти за пределы папки/сети без явного разрешения
  • AGENTS.md — аналог CLAUDE.md (часто делают симлинк, чтобы оба агента читали один файл)
  • GPT-модели — другая «голова», иногда сильнее в одних задачах, слабее в других
  • OpenAI Plus/Pro — отдельная подписка, не съедает Claude-лимиты

Минусы:

  • Подписка отдельная ($20-200/мес у OpenAI) — если уже есть Claude, дублирование
  • Меньше готовых интеграций (skills, hooks — пока проще, чем у Claude Code)
  • Документация рваная — экосистема младше

Кому подойдёт:

  • Тем, кто упирается в лимиты Claude и не хочет на следующую неделю простоя
  • Тем, кто хочет двойной разум — спросить две модели на одну задачу и сравнить
  • Тем, кому важен OS-уровневый sandbox (на боевых машинах)

Cursor (Cursor)

Что это: Полноценный IDE (форк VS Code) со встроенным AI-агентом. Открываете проект, как в редакторе, и работаете в нём же.

Плюсы:

  • Самый комфортный UX для тех, кто привык к VS Code/JetBrains
  • Composer — режим, где агент работает над несколькими файлами сразу
  • Tab-completion AI — дописывает код по ходу набора (это тот самый «магический» опыт)
  • Reject/Accept на каждое изменение — удобный визуальный контроль
  • .cursor/rules/ — правила для агента, привязанные к проекту

Минусы:

  • Подписка ($20+/мес отдельно от Claude/OpenAI)
  • Зависим от одного приложения — если упал Cursor, не работаете
  • Запутать новичка проще — много кнопок, режимов

Кому подойдёт:

  • Тем, кто уже работает в VS Code и не хочет терминал
  • Тем, кому важно видеть код перед глазами все время
  • Тем, кто делает большие проекты с десятками файлов

Менее известные, но достойные

Windsurf (Codeium)

IDE-агент, конкурент Cursor. Сильная сторона — Cascade (агент, который понимает контекст всего проекта). Хорош в больших монорепо. По UX чуть «попроще» Cursor.

Aider (open-source)

CLI, опенсорс. Подключается к любой модели (Claude, GPT, локальная). Минималистичный, но мощный для тех, кто хочет полную свободу. Не для совсем нулевого новичка.

Cline (open-source, плагин для VS Code)

Бесплатный плагин, который превращает VS Code в подобие Cursor. Подключаете свой ключ Claude/OpenAI. Отличный вариант, если не хочется отдельной подписки Cursor.

Моя личная связка

По состоянию на май 2026, у меня:

  • Claude Code Max — основная работа, 80-90% времени
  • Codex CLI Plus — когда упёрся в лимиты Claude (бывает на больших проектах ближе к вечеру) или для второго мнения
  • Cursor — для длинных рефакторингов, где удобно видеть diff
  • Claude.ai в браузере — для чисто разговорных задач (планы, объяснения, идеи)

Это не «правильно» — это то, что мне удобно. У вас может быть совсем другое.

Принципиально: не надо собирать «зоопарк» сразу. Начните с одного. Когда упрётесь в его ограничения — поймёте, какой второй вам нужен. Не идеально, но реально работает.

Как выбрать первый

Если новичок и хочется быстрый старт → Claude Code. Самые мягкие defaults, лучшая onboarding-документация, не надо настраивать.

Если уже привык к VS Code и не хочется в терминал → Cursor (или Cline как бесплатная альтернатива).

Если есть OpenAI Pro и не хочется Anthropic → Codex CLI. Работает похоже, экосистема активно растёт.

Если бюджет 0 рублей → Cline + бесплатный API-ключ (например, через free-tier Anthropic API). Это не «прям удобно», но рабочая связка для обучения.

Как добавить второй (когда время придёт)

Сигнал, что пора добавить второй инструмент:

  1. Регулярно упираетесь в лимиты одного и теряете час «потому что закончились запросы»
  2. Заметили слабую сторону — например, ваш агент плохо в SQL, а другой — отлично
  3. Хочется чего-то, чего нет — например, OS-sandbox или Composer-режима

Тогда поставьте второй, держите один CLAUDE.md / AGENTS.md (через симлинк) на проект, и используйте под задачу.

Practice (20 минут)

Задание 1 (10 минут) — оценить свой workflow

После пары недель работы с Claude Code из B2-B7, ответьте себе на три вопроса:

  1. Где мне было неудобно? (Терминал? Лимиты? Что-то ещё?)
  2. Что я бы хотел иметь? (Подсказки в редакторе? Sandbox? Второе мнение?)
  3. Какой инструмент закрывает именно это?

Если ответы — «всё ок» — отлично, не плодите сущности. Один Claude Code хватает огромному большинству.

Задание 2 (10 минут) — попробуйте альтернативу

Поставьте один альтернативный инструмент (Cursor или Cline или Codex CLI). Сделайте в нём ту же задачу, что в B5 (todo-приложение). Сравните:

  • Что было удобнее?
  • Где запутался?
  • В каком инструменте сам код получился чище?

Запишите свои выводы — это и есть ваш персональный benchmark.

Recap

  • Большая тройка: Claude Code (CLI), Codex CLI (CLI), Cursor (IDE). Плюс достойные альтернативы (Windsurf, Aider, Cline).
  • Один инструмент покрывает 80%+ задач новичка. Не плодите зоопарк заранее.
  • Сигнал добавить второй: лимиты, слабые стороны, конкретная фича, которой не хватает.

Вайб Самурай

🥷 Вайб Самурай спрашивает: «А почему вы советуете Claude Code, а не самый дешёвый/бесплатный? Это реклама?»

Ответ: Если бы у Anthropic было бесплатно — я бы советовал бесплатно. Сейчас Claude Code — это лучший onboarding для нулевого новичка из платных, и Cline — лучший бесплатный. Я искренне работаю на Claude Max ($100/мес), потому что окупается в первую неделю — для моей работы. Для вашей учёбы — Pro ($20) с головой, или вообще бесплатный Cline на free-tier API. Никакой рекламы, я не сотрудничаю ни с одной из этих компаний. Если завтра выйдет инструмент, который очевидно лучше — буду советовать его. Не идеально, но реально работает — выбирайте под бюджет и удобство, не под бренд.

Дальше

В B9 — Деплой первого проекта разберём, как сделать так, чтобы ваш todo-приложение (или любой другой проект) стал доступным в интернете по ссылке. Без серверов, без админки — через GitHub Pages или Cloudflare Pages, бесплатно, 15 минут до результата. Это тот самый момент «у меня есть ссылка, я могу её скинуть другу».

Если зашло — поделитесь главой с тем, кто застрял на одном инструменте и боится попробовать другое. Если у вас уже сложилась своя связка — напишите в комменты, обсудим. За лайк и комментарий — отдельная благодарность, ваша обратная связь реально помогает шлифовать материал.
