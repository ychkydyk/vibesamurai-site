Advanced Path · a20 EN

A20 · Capstone: запуск AI-студии

Сборка: 1 бренд-кейс + 1 СММ-кейс + 1 медиа-инсталляция + 1 RAG-инструмент. Контракт с клиентом, KP / review-сайт, метрики. Финал Advanced.

Что строим

Финальный проект Advanced-трека: запуск рабочей AI-студии за 4 недели. Не «учебный пример», а минимально полная конфигурация, способная принять первый клиентский заказ. Капстоун собирает темы A1–A19 в один воркфлоу.

Цель и периметр

Через 4 недели после старта у студии есть:

  1. Один бренд-кейс — собственный или клиентский бренд с design system (A4), сайт + КП.
  2. Один СММ-кейс — серия из 12 hero + motion-постов в фирменном стиле (A5–A8).
  3. Одна медиа-инсталляция — работающий стенд (A9–A12), готовый к публичному показу.
  4. Один RAG-инструмент — приватный архив с поиском под собственные нужды (A13).

Плюс — операционная база: tokenomics gate (A17), eval (A18), security checklist (A19), local LLM (A14), MCP-админка (A16).

Дорожная карта 4 недель

Неделя 1 — фундамент

  • brand-tokens.json (A4), design system в Figma, code connect.
  • Локальный Qwen на 127.0.0.1:8080 (A14).
  • RAG для собственного архива (A13) — TG + файлы + аудио.
  • MCP-админка + security_monitor (A16, A19).
  • Claude-process-allowlist + claude_managed_ports.

Результат недели 1: инфраструктура готова, бренд формализован, личный архив доступен через RAG.

Неделя 2 — визуальная продукция

  • Pipeline image (A5) — FLUX 2 + бренд-prefix + post-validator.
  • Pipeline marketplace cards / product photoshoot (A6) — если применимо к бренду.
  • Pipeline video (A7) — Seedance 2.0 для motion-pin.
  • Если бренд использует лицо — Soul ID training (A8).
  • Multi-modal orchestrator (A2) с кэшем и lineage.

Результат недели 2: 12 hero + 12 motion-pin для СММ; стоимость на отчёте; все артефакты с lineage.

Неделя 3 — медиа-инсталляция

  • TouchDesigner проект (A9) + StreamDiffusion если применимо.
  • Audio-reactive слой (A10) на VB-Cable + analyzer.
  • Если стенд интерактивный — STT + LLM + TTS + CV (A11).
  • Telemetry → control loop (A12).
  • Security predeploy checklist (A19), demo-токен.

Результат недели 3: работающий стенд, готов к публичному показу 30+ минут без рестартов.

Неделя 4 — клиент-готовность

  • Eval pipeline (A18) для отбора финальных вариантов.
  • Tokenomics gate (A17) во всех узлах.
  • Шаблон клиент-ревью (review-галерея на Cloudflare Pages + Worker + KV) развёрнут.
  • KP / договор / лицензии генеративки проверены.
  • One-click commands для развёртывания под нового клиента.

Результат недели 4: студия готова принять первый клиентский заказ.

Контракт с клиентом

Базовый контракт по итогам Advanced:

- объём работ                      перечень артефактов (count, format, ar, длительность)
- бренд-токены                     ссылка на brand@version
- лицензии генеративки             что генерим, где разрешено коммерч.
- watermark / IP / лица            кто согласовывает, какие референсы запрещены
- ревью                            кол-во раундов, способ (review-галерея с лайк/комментариями)
- сроки                            календарные, не «когда получится»
- стоимость                        генеративка + работа + buffer 15%
- собственность артефактов         клиент / студия

Без формального контракта проект быстро растёт в скоупе и стоимости. Это правило не из «обучения», это из практики.

Метрики студии

После запуска — что мерим, чтобы понимать здоровье:

МетрикаЦель
Среднее время от брифа до 1-го варианта1–2 рабочих дня
Среднее число итераций≤3
Генеративка $ / проектв рамках сметы ± 15%
Локальный vs cloud LLM split≥60% локально
Доля отклонённых клиент-ревью<30%
Время на повтор проекта<50% от первого

Метрики не «для презентации», а для собственной операционной картины. Если повтор проекта не дешевле первого — каскад/кэширование сломаны.

Реальный кейс: связка проектов из практики

Пример работающего портфеля (взаимодействующие проекты из memory):

  • Бренд-проект «Atlas Studio» (см. A4) — design system as code, шаблон KP/landing/review-сайтов; единый brand-tokens.json потребляется кодом, Figma и промпт-шаблонами.
  • Review-галерея (см. A7) — статика на Cloudflare Pages + Worker + KV под клиент-ревью с лайками и комментариями; клонируется под нового клиента за 30 минут.
  • 3 motion-вариации под клиента (см. A7) — применение review-галереи: hero FLUX 2 + 3 варианта Seedance 2.0 image-to-video.
  • Identity-pipeline артиста-перформера (см. A8) — Soul ID на 20 фото + серия публикаций (обложки, motion-pin, live-performance render); ~$3 fixed cost растворяется в выходе.
  • Стек интерактивных инсталляций (см. A11) — три класса (голосовой киоск, AI-витрина, ИИ-зеркало) на одной архитектурной базе STT+LLM+TTS+CV.
  • Приватный RAG-архив (см. A13) — мессенджер-каналы + файлы + аудио через BGE-M3 + LanceDB + Sonnet 4.6 как операционный инструмент студии.

Это не «портфолио для красоты», это вариант того, как темы Advanced связываются в полноценную студию.

Анти-паттерны

  • «Сначала сайт студии, потом первый клиент» — обратно: первый клиент, потом сайт. Сайт без кейсов = пустой.
  • «Возьму проект подешевле для портфолио» — низкий чек = низкий приоритет у вас же. Лучше один проект за полную цену.
  • «Соберу всё сам без агента» — Advanced как раз про то, что собирает агент-orchestrator. Соберёте сами — потеряете скорость.
  • «Когда вырасту — добавлю security» — security добавляется в день 1, не «потом».

Что Advanced не даёт

Честный список того, чего в этом курсе нет (на 2026-05-29):

  • Продажи и маркетинг — не входит в технический скоуп Advanced. Учится отдельно.
  • Юр. оформление — самозанятый/ИП/ООО, ВЭД, налоги. Локально-специфично, в курсе только базовый чек-лист.
  • HR и масштабирование команды — Advanced рассчитан на 1–3 человек на старте.
  • Конкретные тарифы клиентам — рынок волатилен. Калькулятор — да, тарифы — нет.

Эти темы — за рамками технического трека. Возможно, отдельный материал в будущем.

Финал Advanced

Если вы прошли A1–A19 и собрали капстоун — вы:

  • разнесли стек между локальным и облачным inference со вкусом;
  • умеете считать стоимость каждого узла пайплайна и видите драйверы;
  • знаете как делать креативный output повторяемого качества, не «угадывать кадр»;
  • умеете поднять и проектировать медиа-инсталляцию с замкнутой control loop;
  • ведёте RAG над собственным архивом;
  • работаете с MCP — своими и чужими;
  • держите security предсказуемо.

Дальше — только реальные проекты. Курс — это карта. Опыт — годы работы. Берёте первого клиента, проходите через все пункты A1–A19 на боевом проекте, делаете выводы. Через 6–12 месяцев у вас собственный взгляд на то, что в Advanced устарело и что недосказано — пишите фидбек, курс обновляется по таким письмам.
