A20 · Capstone: launching the studio

Assembly: 1 brand case + 1 SMM case + 1 media installation + 1 RAG tool. Client contract, KP / review site, metrics. Advanced finale.

What we’re building

The final project of the Advanced track: launching a working AI studio in 4 weeks. Not a “training example” — the minimally complete configuration ready to take its first paying client. The capstone collects topics A1–A19 into one workflow.

Goal and perimeter

Four weeks after start, the studio has:

  1. One brand case — own or client brand with design system (A4), site + KP.
  2. One SMM case — a series of 12 hero + motion posts in brand style (A5–A8).
  3. One media installation — a working booth (A9–A12), ready for public presentation.
  4. One RAG tool — a private archive with search for your own purposes (A13).

Plus the operating base: tokenomics gate (A17), eval (A18), security checklist (A19), local LLM (A14), MCP admin (A16).

4-week roadmap

Week 1 — foundation

  • brand-tokens.json (A4), design system in Figma, code connect.
  • Local Qwen on 127.0.0.1:8080 (A14).
  • RAG for your own archive (A13) — TG + files + audio.
  • MCP admin + security_monitor (A16, A19).
  • Claude-process-allowlist + claude_managed_ports.

Week 1 result: infrastructure ready, brand formalised, personal archive available via RAG.

Week 2 — visual production

  • Image pipeline (A5) — FLUX 2 + brand prefix + post-validator.
  • Marketplace cards / product photoshoot pipeline (A6) — if the brand applies.
  • Video pipeline (A7) — Seedance 2.0 for motion pins.
  • If the brand uses a face — Soul ID training (A8).
  • Multi-modal orchestrator (A2) with cache and lineage.

Week 2 result: 12 hero + 12 motion pins for SMM; cost on the receipt; every artifact with lineage.

Week 3 — media installation

  • TouchDesigner project (A9) + StreamDiffusion if applicable.
  • Audio-reactive layer (A10) on VB-Cable + analyzer.
  • If the booth is interactive — STT + LLM + TTS + CV (A11).
  • Telemetry → control loop (A12).
  • Security predeploy checklist (A19), demo token.

Week 3 result: working booth, ready for a 30+ minute public showing without restarts.

Week 4 — client-readiness

  • Eval pipeline (A18) for selecting final variants.
  • Tokenomics gate (A17) across all nodes.
  • Client review template (flag-review) deployed.
  • KP / contract / generative licences reviewed.
  • One-click commands for deploying for a new client.

Week 4 result: the studio is ready to take its first paying client.

Client contract

A base contract after Advanced:

- scope of work               list of artifacts (count, format, ar, length)
- brand tokens                link to brand@version
- generative licences         what we generate, where commercial use is allowed
- watermark / IP / faces      who signs off, which refs are forbidden
- review                      number of rounds, method (flag-review)
- deadlines                   calendar, not "when ready"
- cost                        generative + work + 15% buffer
- artifact ownership          client / studio

Without a formal contract, scope and cost balloon fast. This rule isn’t from “training,” it’s from production.

Studio metrics

After launch — what to measure to gauge health:

MetricTarget
Avg time from brief to first variant1–2 business days
Avg number of iterations≤3
Generative $ / projectwithin budget ± 15%
Local vs cloud LLM split≥60% local
Share of rejected client reviews<30%
Time to repeat a project<50% of first-time

Metrics aren’t “for the presentation,” they’re for your operating picture. If a repeat project isn’t cheaper than the first — the cascade/caching is broken.

This isn’t “portfolio for show” — it’s one way Advanced topics tie into a full studio.

Anti-patterns

  • “Studio site first, then the first client” — reverse: first client, then site. A site without cases is empty.
  • “I’ll take a cheaper project for the portfolio” — low cheque = low priority on your own side. Better one project at the full rate.
  • “I’ll build it all myself without an agent” — Advanced is about what an orchestrator agent assembles. Build it yourself and you lose the speed.
  • “I’ll add security when I grow” — security is day one, not “later.”

What Advanced doesn’t cover

An honest list of what’s not in this course (as of 2026-05-29):

  • Sales and marketing — outside the technical Advanced scope. Learned separately.
  • Legal setup — self-employed/IP/LLC, cross-border, taxes. Locale-specific; only a base checklist in the course.
  • HR and team scaling — Advanced is sized for 1–3 people at launch.
  • Specific client tariffs — the market is volatile. Calculator — yes; tariffs — no.

These topics are outside the technical track. Possibly a separate material in the future.

Advanced finale

If you’ve worked through A1–A19 and assembled the capstone, you:

  • split the stack between local and cloud inference with taste;
  • can compute the cost of every pipeline node and see the drivers;
  • know how to produce creative output of repeatable quality, not “guess the right frame”;
  • can stand up and architect a media installation with a closed control loop;
  • run RAG over your own archive;
  • work with MCPs — your own and others’;
  • keep security predictable.

From here — only real projects. The course is a map. Experience is years of work. Take the first client, run them through every A1–A19 stage on a live project, draw the lessons. In 6–12 months you’ll have your own view of what in Advanced is stale and what’s understated — write feedback, the course updates from those letters.
