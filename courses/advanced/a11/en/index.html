A11 · Interactive booths: STT+LLM+TTS+CV

Architecture for an interactive booth. Local Qwen + Fish Speech + camera, cloud-endpoint queues, demo tokens, cached TTS. Cases: Tsoy/fitting/PMEF.

What we’re building

An interactive booth: the user speaks → STT → LLM composes a reply → TTS voices it + UI/CV visuals react. Latency “question → answer” 2–6 seconds. The architecture has a local fallback for when cloud endpoints fail and uses no public ports (rule feedback_no_wildcard_http_server).

Base layout

[mic] ─► STT ─► text ─► LLM ─► response_text ─► TTS ─► [speakers]
                              │
                 [camera] ─► CV ─► attributes ─► LLM context
                                                          │
                                                          ▼
                                                     UI/visuals
                                                     react

CV input (optional): age band / mood / count of people in frame. Those attributes are passed into the LLM as context, so the reply matches the audience.

Principle: everything on 127.0.0.1, no public ports. No Cloudflare Tunnel or public host (memory project_fitting_kiosk_local_only).

Fish Speech on port 7870

From production (memory project_tsoy_fish_port): Fish Speech runs on 127.0.0.1:7870 because port 8080 is taken by the local Qwen llama-server. The VSCode plugin gates the Fish probe by web=ok — check before start.

import requests

def tts(text: str, voice: str = "tsoy") -> bytes:
    resp = requests.post(
        "http://127.0.0.1:7870/v1/tts",
        json={"text": text, "voice": voice},
        timeout=30,
    )
    resp.raise_for_status()
    return resp.content

TTS cache as RAG material

Rule (memory feedback_tts_cache): every TTS reply is saved to data/tts_cache/<voice>/ with the question metadata — for future RAG vectorisation.

data/tts_cache/
├── tsoy/
│   ├── 9f3a1c.wav
│   ├── 9f3a1c.json        ← {question, response_text, ts, voice, model}
│   ├── 4b2e8d.wav
│   └── 4b2e8d.json
└── duremar/
    └── ...

Double benefit: (1) a repeat question is served instantly from the cache without another TTS call; (2) you accumulate a “question → reply” dataset that later powers RAG and quality evaluation.

Cloud endpoints: queue check

Rule (memory feedback_runpod_queue_check + feedback_cloud_activation_order): before a cloud start and after any restart/stop check inQueue / inProgress for the LLM and TTS endpoints. On cloud activation — bring TTS and LLM up in parallel, call TTS first (it warms slower).

def check_endpoint(url: str) -> dict:
    r = requests.get(f"{url}/health", timeout=5).json()
    return {
        "in_queue": r.get("workers", {}).get("queued", 0),
        "in_progress": r.get("workers", {}).get("inProgress", 0),
        "ready": r.get("workers", {}).get("ready", 0),
    }

# Cloud start sequence
threads = [
    Thread(target=warm_up, args=(tts_url,)),  # first
    Thread(target=warm_up, args=(llm_url,)),  # parallel, starts later
]

Without a queue check the booth stalls for minutes while workers are still warming up.

Demo token for client presentations

From production (memory project_tsoy_demo_token): /demo?demo_token=… enables a seamless client demo without a Basic-Auth popup. Accept the token via query, then proceed with normal session auth.

@app.get("/demo")
def demo(demo_token: str):
    if not constant_time_eq(demo_token, settings.DEMO_TOKEN):
        return RedirectResponse("/")
    response = RedirectResponse("/booth")
    response.set_cookie("session", make_session_token(), httponly=True)
    return response

One token in .env, one query param, clean presentation. For a client demo it’s a must.

Real cases

tsoy_booth (memory project_tsoy_booth)

An AI voice booth of Viktor Tsoi: the user asks a question → “Tsoi’s voice” replies (Fish Speech on a voice dataset) + text. Local Qwen for response generation, Fish Speech on 7870. The TTS cache grows, so over time repeat questions come back instantly.

Security: a scheduled task TsoyBoothSecurityCheck (memory project_tsoy_security_task) — runs every 30 minutes, via pythonw with no window. Works independently of the booth.

Mandatory pre-deploy security checklist (memory feedback_security_predeploy) before every deploy.

fitting_kiosk (memory project_fitting_kiosk)

An AI shop-window fitting room: ComfyUI + TouchDesigner. Dev on a 4070 8 GB + 64 GB RAM, prod target — 5090. Local-only on 127.0.0.1, no auth needed for admin endpoints. The ComfyUI terminal launches via Start-Process run_kiosk.bat (visible cmd window — rule feedback_comfyui_visible_terminal).

pmef_booth (memory project_pmef_booth_mvp)

An AI mirror for the SPIEF (St. Petersburg International Economic Forum): CV input (6 archetypes from content/archetypes.json) → selected reply → visual. MVP — static HTML; next — spec/{API,METRICS,WS_PROTOCOL}.md for the CV contractor. Deadline: May 25.

CV context for the LLM

A simple example of pushing CV attributes into the LLM’s system prompt:

attrs = mediapipe_analyze(frame)  # {"age_band":"30-45","mood":"smiling","people":1}
sys = f"""You are the voice of Viktor Tsoi. React to the interlocutor:
age {attrs['age_band']}, mood {attrs['mood']}, people in frame {attrs['people']}.
Reply short: 1-3 sentences, in the style of his songs."""

The LLM gets context — it doesn’t “guess” the interlocutor. CV here is a context sensor, not a recogniser.

Pitfalls

  • Wildcard bind 0.0.0.0 — forbidden (memory feedback_no_wildcard_http_server). Use --bind 127.0.0.1 only.
  • Cloud not warm — the booth stalls for minutes in production. Warm-up + queue check are mandatory.
  • TTS cache grows without cleanup — after 6 months of operation you have 50 GB of replies. TTL + rotation.
  • Whisper on CPU — slow on long-form audio. For a booth (short phrases) it’s fine; for long audio offload to GPU.
  • Network at the demo venue — a constant pain. A local fallback (no cloud) is mandatory. If the network dies the booth keeps running on local Qwen.

Recap

  • Base stack: STT (Whisper) + LLM (local Qwen / cloud Sonnet) + TTS (Fish Speech) + CV (MediaPipe).
  • 127.0.0.1 only; no public ports, no Cloudflare Tunnel.
  • TTS cache with question metadata — saves money and builds a dataset for future RAG.
  • Cloud activation: TTS first, in parallel with LLM. Queue check before start.
  • CV attributes are pushed into the LLM’s system prompt as context.
  • Demo token for the client presentation; security checklist before every deploy.
