A18 · Eval для креативного output

Virality predictor, hook strength, retention risk, brand fit. Когда оцениваем человеком, когда автоматом. A/B на инсталляции и клиент-ревью.

Что строим

Eval-пайплайн для креативного output: измерение того, что обычно «оценивается на глаз». Цель — повторяемый процесс отбора лучшего варианта из N, а не «глянул сценарий, давайте этот».

Middle (M11–M13) дал базу eval для кода и текста. Advanced — про eval визуального и аудио-видео креатива.

Три уровня оценки

─────────────────────────────────────────────────────────────
авто-метрики   тех. соответствие                скрипт
               (палитра, разрешение, fps,
                пропорции, табу-слова)
─────────────────────────────────────────────────────────────

Уровни — последовательны. Авто-метрики (низкая стоимость) фильтруют технический брак. AI eval сужает топ-N. Человек принимает финал.

Авто-метрики: технический фильтр

Что измеряем без AI:

  • палитра: гистограмма цветов попадает в брендовый диапазон (#0A0A0A ± 10, #C8FF00 ± 20);
  • резкость: Laplacian variance — отсев размытых кадров;
  • пропорции: 4:5 / 9:16 / 16:9 строго;
  • discrepancies: лишние объекты, рекламные плашки (через CLIP-zero-shot);
  • табу-слова в субтитрах: regex.
def auto_check(image_path: Path, brand: dict) -> list[str]:
    img = cv2.imread(str(image_path))
    errors = []
    if avg_color_diff(img, brand["palette"]) > 0.15:
        errors.append("палитра вне бренда")
    if cv2.Laplacian(img, cv2.CV_64F).var() < 50:
        errors.append("кадр размыт")
    h, w = img.shape[:2]
    if abs((w/h) - brand["target_ar"]) > 0.05:
        errors.append(f"пропорции {w/h:.2f}, нужны {brand['target_ar']:.2f}")
    return errors

Авто-метрики отсекают, не «оценивают». Они скажут «технически приемлемо», не «выбери это».

Virality predictor

Из практики (skill higgsfield-generate с brain_activity): Virality Predictor анализирует видео-ролик по нескольким осям:

  • hook strength — насколько первые 1–3 секунды зацепят;
  • attention curve — где взгляд аудитории;
  • retention risk — где зритель потеряет интерес;
  • distraction risk — конкурирующие визуальные элементы;
  • creative score — суммарная оценка.

Используется в A18 как AI-eval уровень: пропускаем top-N клипов через virality, выбираем по hook strength и retention.

/higgsfield-generate brain_activity --video path/to/clip.mp4

Не превращайте эту оценку в финальное решение — это сигнал, не «правда о виральности». Reality check — A/B на реальной аудитории.

Brand fit

Brand fit — отдельный AI-eval: насколько кадр/клип соответствует бренд-токенам. Дешёвый вариант — CLIP с brand-prefix prompt.

import torch, open_clip

model, _, preprocess = open_clip.create_model_and_transforms("ViT-B-32", pretrained="laion2b_s34b_b79k")
tokenizer = open_clip.get_tokenizer("ViT-B-32")

def brand_score(image: Image, brand_prompt: str) -> float:
    img_feat = model.encode_image(preprocess(image).unsqueeze(0))
    txt_feat = model.encode_text(tokenizer([brand_prompt]))
    return torch.cosine_similarity(img_feat, txt_feat).item()

brand_prompt — короткое описание визуального языка: «editorial high-contrast monochrome with single chartreuse accent, geometric, minimal». Кадры рангируем по score, отсекаем хвост.

A/B на клиент-ревью

Из практики (memory project_flag_review / project_svib_review): клиент-ревью с видео-галереей, лайками и комментариями. Шаблон сайта (flag-review) переиспользуется под каждый проект.

Архитектура:

  • статика на Cloudflare Pages под zavdv1;
  • Worker + KV для лайков и комментариев;
  • 30 минут — клонировать шаблон под новый проект.

Используем как A/B: показываем 3 motion-варианта одного концепта, клиент голосует/комментирует. Решение — на основе фидбека, не на «нравится же».

Старые генерации — не удаляем, переносим в архив (data-default-archived="true") — правило feedback_archive_never_delete. Прозрачность истории дизайна важна.

Реальный кейс: 12 постов → выбор 4 финалов

Постановка: серия из 12 hero-кадров (см. A5). Выбрать 4 финала для публикации в течение месяца.

Воркфлоу:

  1. Авто-метрики на 12 кадров → 1 отсев (Laplacian < 50, размыт). Остаётся 11.
  2. Brand fit через CLIP → ранг top-8 по score > 0.32. Остаётся 8.
  3. Virality predictor на mp4-превью каждого → ранг по hook strength. Топ-5.
  4. Клиент-ревью: показываем 5 в flag-review, клиент комментирует. Из 5 выбирается 4 для публикации.

Время на отбор: 1 час (большая часть — клиентское ревью). До eval-пайплайна — 3–4 часа «на глаз» с переписками.

ЧтоКтоПочему
Тех. соответствие (пропорции, цвет, резкость)СкриптОбъективно, дёшево

Не пытайтесь автоматизировать «художественную композицию» — AI eval показывает паттерны, не вкус. Не пытайтесь оценивать «hook» на глаз — virality predictor дешевле и в среднем точнее.

Подводные камни

  • Virality eval ≠ виральность — это сигнал. Хорошо работает на сравнении вариантов, плохо — как абсолютная метрика.
  • CLIP brand-fit на ProMpt-prefix плывёт — если brand prefix сменился между измерениями, score-ы несравнимы. Версионируйте prompt.
  • Клиент голосует за «как тот ролик» — субъективные предпочтения мешают системному отбору. Структурируйте опрос (по осям hook / brand / message).
  • «У нас X лайков» — лайки на ревью-сайте — это не метрика виральности, это вовлечённость 5–10 человек. Не выдавайте за «прогноз».

Recap

  • Три уровня: авто-метрики (фильтр) → AI-eval (ранжирование) → человек (финал).
  • Virality predictor — сигнал на сравнении, не «правда о виральности».
  • Brand fit через CLIP — масштабируемая оценка соответствия бренду.
  • A/B клиент-ревью — flag-review шаблон, 30 минут под клиента.
  • Старые варианты не удаляем — архивируем.
