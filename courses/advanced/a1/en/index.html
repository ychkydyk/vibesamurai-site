A1 · AI studio stack — May 2026

Modality × tool × case matrix. Where agents write code, where they orchestrate services. Anchor for the entire Advanced track.

What we’re building

A map of the stack an AI studio actually runs in May 2026. Not a “list of favourite tools” but a decision table: which engine for which task, what a unit of output costs, what the limits are. The map is the anchor for the remaining 19 Advanced chapters.

Two agent roles in a studio

In a 2026 studio an agent works in two modes. Don’t conflate them — the tooling, the metrics, and the risks are different.

RoleWhat it doesArtifactWhere the difficulty is
Code agentWrites code, fixes bugs, scaffolds projectsDiff, PR, binaryCorrectness + review
Pipeline orchestratorConducts generative services (image/video/audio/CV)File/frame/sequence/installationCost, latency, idempotency

Code agent is what you mastered in Beginner and Middle (Claude Code, Codex, Cursor). Pipeline orchestrator is Advanced’s specific concern: you’re not writing the code, you’re writing the program that calls FLUX, Seedance, Higgsfield, RunPod, TouchDesigner.

In a real studio both roles sit side by side: the code agent writes the orchestrator that calls generative services.

Choosing a tool

There’s no “universally best” engine. Decide along three axes:

  1. Cost per unit of output — image and video at scale are orders of magnitude more expensive than LLM.
  2. Latency — an installation needs <1 s; SMM creative tolerates minutes.
  3. Privacy — the private archive and client data don’t leave for the cloud.

Real cases from the industry

To keep the stack map from staying abstract — four publicly documented works by reference studios in generative art and media installations.

Refik Anadol Studio — “Unsupervised — Machine Hallucinations: MoMA” (2022–2023)

A large-scale audio-visual installation at the Museum of Modern Art in New York. Anadol’s studio trained the StyleGAN family on MoMA’s collection dataset (~138,000 works) and ran real-time inference on a ~7×7 m LED wall. Audio was synchronised with visuals via OSC. Six months of continuous exhibition with automatic worker restart.

Lesson: for a long-running show uptime is engineered explicitly — the system must restart itself without an operator. This is the topic of modules A11–A12: telemetry → control loop, heartbeat.

TeamLab — Borderless (Tokyo / Shanghai / Jeddah)

The Japanese collective TeamLab operates permanent exhibitions in several cities. Inside — a mesh of hundreds of GPU stations synchronised into one scene via TouchDesigner and Spout. The AI component: CV-based recognition of visitor movement plus generative visuals reacting to audience behaviour.

Lesson: this is a distributed real-time system. One powerful machine isn’t enough — you need a network of coordinated nodes. Modules A9 (TD + StreamDiffusion) and A12 (telemetry) cover this class of architecture in detail.

OUCHHH — “Poetic AI” at Atelier des Lumières (Paris)

The Turkish studio projected AI-generated visuals trained on a corpus of Mallarmé, Baudelaire, and Rimbaud onto the walls and floor of the Atelier des Lumières space in Paris. The venue uses around 140 projectors with overlap zones, and the media server aligns the frames. The ML model outputs frames in a format compatible with the media server.

Lesson: the physical layer constrains the ML stack. 140 projectors with overlap require a separate calibration and alignment system. The model can’t “generate however it likes” — the venue dictates the picture format.

Random International — “Rain Room” (Barbican / MoMA / LACMA / Sharjah)

The British studio Random International. A performative installation in which the rain physically stops above a visitor. The technical layer: depth cameras track bodies, the control system shuts off corresponding sections of the water grid in the ceiling.

Lesson: AI in an installation isn’t necessarily generative. The main thing here is a CV layer (tracking people) and a control loop with a physical effector (solenoids regulating water flow). This class of work opens an entire direction where the LLM only appears at the prep stage — for documentation, application, technical rider. Modules A11 (interactive booths) and A12 cover this.

The common skeleton

All four cases use the same architecture:

  1. Pipeline orchestrator (often TouchDesigner for real-time, Python for data prep).
  2. AI model for the task — generative (Anadol, OUCHHH) or discriminative (Random, TeamLab CV).
  3. Physical layer — projectors / LED, sensors (RGB, depth), audio output, optionally DMX-controlled light and physical actuators.
  4. Control loop with heartbeat monitoring and automatic restart.

This is the minimum architecture for an installation of the “permanent exhibition” class in 2024–2026.

On your own scale: four profiles

At reference studios the stack looks similar, but the load profiles differ. The same skeleton scales down like this:

Creative agency of 5–8 people

Composition: 2 creative directors, 2 designers, 1 motion designer, 1 techno-artist, 1 producer. They produce immersive campaigns for brands — projection mapping for product launches, immersive stands at fairs, AR filters.

Stack: FLUX 2 + Nano Banana 2 (stills), Seedance 2.0 + Kling 3.0 (offline motion), TouchDesigner + StreamDiffusion (real-time for stands), Soul ID (recurring characters), Sonnet 4.6 + Opus 4.7 (copy).

Monthly: ~$400–700 on generative. Capex on hardware: $15–30k (short-throw 5000–7000 lm projector such as Epson EB-PU2200B, RealSense D455, workstation with RTX 4090).

Solo media artist preparing open-call submissions

One person with an authorial practice. Prepares submissions to Ars Electronica, transmediale, NODE Forum, MUTEK, Lumen Prize. Monthly volume — one or two full portfolio videos plus render documentation.

Stack: FLUX 2 + GPT Image 2 (visuals for documentation and application forms), Seedance 2.0 + Higgsfield Soul Cinema (cinematic teaser), TouchDesigner + StreamDiffusion (installation prototype on a 4070), Opus 4.7 (artist statement — linguistic precision matters), FLUX LoRA Fast (consistency between documentation frames).

Monthly: ~$80–150 on generative. Capex: $5–10k (BenQ TH685 or similar 3000-lm projector, Kinect Azure or RealSense, station with RTX 4070/4080).

AV studio / VJ collective of 2–5

Live visuals for events (concerts, festivals, club nights). Tour-ready setup.

Stack: TouchDesigner as the core, StreamDiffusion + FLUX schnell (AI-augmented visuals), Resolume Arena (classic VJ practice), FLUX 2 (scene prep), Runway Gen-4 Unlimited (loop video for sets), Soul ID (recurring artist), Ableton Live + audio-reactive CHOPs in TD.

Monthly: $200–500 on generative (most real-time runs on owned hardware). Capex: $20–40k (1–3 projectors 8000+ lm for concert stage, MIDI controller, RME Fireface UCX II or Focusrite Scarlett 18i20, laptop with RTX 4090).

Resident engineer at a gallery or museum

One in-house technical specialist. Tasks: maintenance of permanent installations, mounting/dismounting of temporary exhibitions, technical riders for invited artists, documentation.

Stack: TouchDesigner as a universal “glue,” MediaPipe + depth cameras (touchpoints and visitor tracking), Sonnet 4.6 (technical documentation and maintenance log), Claude Code (watchdogs, autorun scripts, projector calibration utilities), LoRA training of the institution’s style.

Monthly: $200–400 on generative. Capex: institution budget — 10,000+ lm projectors of the Christie / Panasonic / Barco class, media-server machines (Modulo Player, Watchout), a network of DMX/Art-Net controllers (MA Lighting, ETC), multichannel audio (RME MADIface XT).

Audio interfaces

For multichannel sound in installations — RME Fireface UCX II (8 in / 8 out), Focusrite Clarett+ 8Pre, or MADI systems (RME MADIface XT) in distributed setups. For VJ work, a Focusrite Scarlett Solo stereo input is enough: it taps the main signal from the master bus of the mixer.

Lighting control — DMX-512 and Art-Net

Lights in installations are controlled through industrial protocols:

  • DMX-512 — the classic 5-pin XLR protocol, up to 512 channels per universe.
  • Art-Net / sACN — network protocols on top of Ethernet, carrying multiple DMX universes. Supported by TouchDesigner out of the box and by most media servers.

Controllers — MA Lighting / ETC consoles; software — Lightkey, Q LAB. Fixtures — Robe, Chauvet Professional, ETC Source Four. For LED walls there’s a separate class (Absen, ROE Visual).

What they expect on an open call

A minimum application pack for an AI/installation work:

  1. Artist statement (300–500 words) — concept of the work, context in the artist’s practice.
  2. Project description (500–1000 words) — technical description, what the work demonstrates, risks.
  3. Documentation video (2–5 minutes) — demo cut + interview + behind-the-scenes.
  4. Technical rider — equipment list (owned / requested), spatial requirements, power, cable infrastructure.
  5. Budget breakdown — what production funds will be spent on if granted.
  6. CV and portfolio — links to previous work.

Claude Opus 4.7 is well-suited for polishing the artist statement — linguistic precision matters. But the artist writes the draft themselves: festival curators read AI-generated text “at a glance” and treat it coolly. AI is useful in preparing the documentation video (see A5, A7, A8) and when assembling the technical rider — Claude Code can produce a neat markdown to a template.

Pitfalls

  • “We have GPT, it can do everything” — no. At scale, the gap between specialised engines and universal LLMs is 10–100× in cost and quality.
  • “Local is cheaper” — only if you have free GPU time and don’t need a fresh model. At volume cloud Batch API often beats your own hardware once you amortise the silicon.
  • “We’ll pick tools as we go” — no. Lock the stack map for a project before you start, or you’ll burn budget benchmarking in production.
  • “We’ll buy hardware for this specific project” — capex on projectors and sensors only pays off with repeated use. For a single-shot project, rent from professional rental companies (AV Stumpfl, PRG, Eclipse Staging Services).
  • “An AI-generated artist statement saves time” — festival curators spot it in a minute. The final text is authorial; AI is admissible as a style editor.
  • Copying a big player’s stack one-to-one — Refik Anadol Studio’s stack only makes sense at its production volume; for a solo media artist it’s overengineering and a cash gap.
  • Engine versions change every 1–3 months — re-review the map before each new project. Source: vendor changelogs + arena leaderboards (image, video, code).

Recap

  • The agent in a studio is either a code agent or a pipeline orchestrator; in installation practice a third role is added — the on-site operator.
  • Stack map = “modality × tool × cost × case” table plus the physical layer (projection / sensors / audio / light). Without it, the budget isn’t controlled.
  • Decision along three axes: cost per unit, latency, privacy.
  • Reference studios (Refik Anadol Studio, TeamLab, OUCHHH, Random International) and small teams use the same skeleton — what differs is the load profile, capex, and share of code-agent work.
  • Open calls (Prix Ars Electronica, transmediale, MUTEK, NODE Forum, Lumen Prize, residencies) are the main entry channel into the industry.
  • Re-review the map before each project; vendor prices and model versions shift every 1–3 months.
