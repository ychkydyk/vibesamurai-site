A7 · Video generation pipelines

Seedance 2.0, Kling 3.0, Runway Gen-4, Higgsfield Soul Cinema. Когда какой движок, multishot JSON для glitch/datamosh, очередь Runway-Unlimited, fallback-стратегии.

Что строим

Очередь генерации видео под СММ и клиентов. Каждый ролик — 5–15 секунд, фирменная стилистика, повторяемый процесс. Пайплайн умеет переходить между движками по правилам (image-to-video, text-to-video, multishot, fallback), уважает бюджет и не дублирует уже сделанные клипы.

Multishot JSON для Seedance

Из практики (memory feedback_seedance_multishot_json): для glitch/datamosh/VHS/reality-collapse и подобных стилей классическая prompt-формула проигрывает multishot JSON. Шаблон:

{
  "shots": [
    {
      "duration": 1.2,
      "camera": "static",
      "subject": "object in clean room",
      "look": "pristine, high-contrast"
    },
    {
      "duration": 0.8,
      "camera": "no movement",
      "subject": "object",
      "look": "first glitch flash, RGB split"
    },
    {
      "duration": 1.5,
      "camera": "no movement",
      "subject": "object dissolving",
      "look": "datamosh, pixel smear, chromatic aberration intensifying"
    },
    {
      "duration": 1.5,
      "camera": "no movement",
      "subject": "object reformed but distorted",
      "look": "VHS bands, magnetic interference"
    }
  ],
  "audio_hint": "static crackle"
}

Без multishot JSON Seedance даёт «однообразный плавный motion» — не то, что просится в задаче. С multishot вы получаете требуемую структурированную динамику.

Runway Unlimited: только Unlimited

Правило студии (memory feedback_runway_unlimited_only): генерации Runway всегда в Unlimited (free), paid-режим — только по явному подтверждению.

Когда Runway оправдан:

  • быстрая итерация: 30 вариантов одного концепта за день;
  • конкретные стили Gen-4 (есть характерный вид, который сложно воспроизвести в Seedance);
  • web-only задачи (UI Runway удобнее API).

Пайплайн runway_queue (memory project_runway_queue) — оркестратор Runway-видео через web-Playwright по умолчанию, Dev API как fallback. Non-aggressive: пауза между задачами, никаких параллельных вкладок.

Реальный кейс: svib-review

Из практики (memory project_svib_review): 3 motion-вариации одного концепта (Крымский мост, под бренд WEDOIT) для клиент-ревью.

Стек:

  • Hero image — FLUX 2 на бренд-токенах (см. A5);
  • 3 варианта motion — Seedance 2.0 image-to-video, image = hero, разные camera-промпты (slow push-in / static / drone arc);
  • Сборка превью — статический сайт по шаблону flag-review (memory project_flag_review) на Cloudflare Pages + Worker + KV для лайков и комментариев;
  • Деплой — svib-review.pages.dev, переиспользует Worker от flag-review.

Время от брифа до ссылки клиенту: 3–4 часа. Стоимость генеративки: ~$8 (3 × ~8 секунд × $0.30 + image hero).

Tokenomics gate (обязательно)

Видео — самая дорогая модальность. Считаем всегда:

def estimate_video(engine: str, clips: int, sec_per_clip: float):
    cost = {"seedance-2": 0.30, "kling-3": 0.40, "soul-cinema": 0.50}
    total = cost[engine] * clips * sec_per_clip
    return f"{engine}: {clips} клипов × {sec_per_clip}с = ${total:.2f}"

print(estimate_video("seedance-2", 12, 6))  # → $21.60
# подтверждение [y/N] обязательно

Особенно перед итерацией multishot: 3 захода × 12 клипов × 6 секунд = $64.80. Без gate бюджет тает быстро.

Fallback-стратегия

Один движок может не взять задачу с первой попытки (модерация, неудачная композиция, артефакты). Прописывайте fallback явно:

Seedance 2.0  ── артефакт / отказ ──►  Kling 3.0 (text-to-video)
Kling 3.0     ── артефакт          ──►  Runway Unlimited (manual review)
Runway        ── artifact          ──►  human regen с правкой промпта

Не бесконечный retry с тем же промптом. Каждый retry — это деньги.

Подводные камни

  • Длительность клипа — Seedance до 10 c, Kling до 10 c, Runway до 16 c. Для 30-секундного ролика — режем на склейки + ffmpeg.
  • 24fps vs 30fps — движки отдают 24/25/30 fps непредсказуемо. Перед сборкой нормализуйте через ffmpeg, иначе плывёт sync с аудио.
  • Image-to-video не сохраняет точно hero — фон/детали поменяются. Если нужна высокая верность hero — закладывайте 1–2 кадра «безопасной» эстетики в начале клипа.
  • Multishot не работает на короткой длительности — суммарно <3 c не стоит дробить на 3 шота. Для коротких — обычный промпт.
  • Runway web flow ломкий — UI меняется, селекторы Playwright плывут. Держите non-aggressive скрипт, регулярно проверяйте.

Recap

  • 4 движка, простое правило выбора по типу входа (text / image / character / glitch).
  • Multishot JSON — обязателен для glitch/datamosh/VHS/reality-collapse. Skill seedance2 хранит шаблоны.
  • Runway всегда в Unlimited, paid — только по явному подтверждению.
  • Tokenomics gate перед партией; fallback прописан явно, не как бесконечный retry.
