A7 · Video generation pipelines

Seedance 2.0, Kling 3.0, Runway Gen-4, Higgsfield Soul Cinema. Which engine when, multishot JSON for glitch/datamosh, Runway-Unlimited queue, fallback strategies.

What we’re building

A video generation queue for SMM and clients. Every clip — 5–15 seconds, on-brand look, a repeatable process. The pipeline knows how to move between engines by rule (image-to-video, text-to-video, multishot, fallback), respects the budget, and doesn’t duplicate clips that already exist.

Multishot JSON for Seedance

From production (memory feedback_seedance_multishot_json): for glitch/datamosh/VHS/reality-collapse, the classic prompt formula loses to multishot JSON. Template:

{
  "shots": [
    {
      "duration": 1.2,
      "camera": "static",
      "subject": "object in clean room",
      "look": "pristine, high-contrast"
    },
    {
      "duration": 0.8,
      "camera": "no movement",
      "subject": "object",
      "look": "first glitch flash, RGB split"
    },
    {
      "duration": 1.5,
      "camera": "no movement",
      "subject": "object dissolving",
      "look": "datamosh, pixel smear, chromatic aberration intensifying"
    },
    {
      "duration": 1.5,
      "camera": "no movement",
      "subject": "object reformed but distorted",
      "look": "VHS bands, magnetic interference"
    }
  ],
  "audio_hint": "static crackle"
}

Without multishot JSON, Seedance returns “uniform smooth motion” — which is not what these tasks call for. Multishot gives you the structured dynamic the brief actually asks for.

Runway Unlimited: Unlimited only

Studio rule (memory feedback_runway_unlimited_only): Runway generations always go through Unlimited (free), paid mode only on explicit confirmation.

When Runway is justified:

  • fast iteration: 30 variants of one concept in a day;
  • specific Gen-4 styles (a distinctive look that’s hard to reproduce in Seedance);
  • web-only tasks (Runway’s UI is more convenient than the API).

A Runway video orchestrator is typically a web-Playwright pipeline by default, with the Dev API as fallback. Non-aggressive: pauses between tasks, no parallel tabs.

Case: 3 motion variants for client review

A typical creative-agency task — show the client 3 variants of one motion concept before production. I’ll illustrate with a generic brief: a brand needs an 8-second animation of an architectural object for the baseline landing series.

Stack:

  • Hero image — FLUX 2 on brand tokens (see A5);
  • 3 motion variants — Seedance 2.0 image-to-video, image = hero, different camera prompts (slow push-in / static / drone arc);
  • Preview assembly — static site templated from a review-gallery skeleton on Cloudflare Pages + Worker + KV for likes and client comments; the template clones to a new client in 30 minutes.
  • Deploy — a subdomain on *.pages.dev, dedicated Worker per review with rate-limiting on likes.

Time from brief to client link: 3–4 hours. Cost of generative for the final Seedance clips: 3 × ~8 seconds × ~$0.27 ≈ $6.5 + ~$0.03 for the FLUX 2 hero (1 MP) ≈ $6.5. If drafts run on Kling 3.0, add small change.

The format “three motion variants on a single page with like/comment threads” is the de facto standard for client-review motion pitches in 2024–2026, used by studios at the level of Field.io, Universal Everything, and ManvsMachine.

Tokenomics gate (mandatory)

Video is the most expensive modality. Compute the cost every time:

def estimate_video(engine: str, clips: int, sec_per_clip: float):
    # Prices verified 2026-05-29
    cost = {
        "seedance-2":  0.27,   # FAL 720p, middle of $0.24–0.30/s
        "kling-3":     0.029,  # sharp drop by May 2026
        "runway-g4-turbo": 0.05,
        "soul-cinema": 0.50,
        "sora-2-pro":  0.40,   # middle of $0.30–0.50
        "veo-3":       0.75,
    }
    total = cost[engine] * clips * sec_per_clip
    return f"{engine}: {clips} clips × {sec_per_clip}s = ${total:.2f}"

print(estimate_video("seedance-2", 12, 6))         # → $19.44
print(estimate_video("kling-3", 12, 6))            # → $2.09  (on iterations)
print(estimate_video("runway-g4-turbo", 12, 6))    # → $3.60
# [y/N] confirmation required

Especially before a Seedance multishot iteration: 3 passes × 12 clips × 6 seconds = $58.32. On Kling 3.0 the same batch = $6.26. That gap is the main driver of engine choice on the draft phase.

Fallback strategy

A single engine may fail to take the task on the first try (moderation, bad composition, artefacts). Make the fallback explicit:

Seedance 2.0  ── artefact / reject ──►  Kling 3.0 (text-to-video)
Kling 3.0     ── artefact           ──►  Runway Unlimited (manual review)
Runway        ── artefact           ──►  human regen with prompt edit

Not infinite retry with the same prompt. Every retry costs money.

Pitfalls

  • Clip length — Seedance up to 10 s, Kling up to 10 s, Runway up to 16 s. For a 30-second clip, chunk into edits + ffmpeg.
  • 24fps vs 30fps — engines return 24/25/30 fps unpredictably. Normalise via ffmpeg before assembly, otherwise audio sync drifts.
  • Image-to-video doesn’t preserve the hero exactly — background and details will shift. If you need high hero fidelity, plant 1–2 “safe-aesthetic” frames at the start of the clip.
  • Multishot doesn’t work on short clips — under ~3 s total, don’t try to split into 3 shots. For short clips use a regular prompt.
  • Runway web flow is fragile — UI changes, Playwright selectors drift. Keep the non-aggressive script and re-test regularly.

Recap

  • 4 engine families, simple selection rule by input type (text / image / character / glitch).
  • Multishot JSON is mandatory for glitch/datamosh/VHS/reality-collapse. Skill seedance2 stores the templates.
  • Runway always in Unlimited; paid only on explicit confirmation.
  • Tokenomics gate before each batch; fallback declared explicitly, not as infinite retry.
