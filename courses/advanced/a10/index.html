A10 · Audio-reactive AV-инсталляции

TD-аудио (VB-Cable loopback), spectral / onset / envelope, маппинг в визуал. 27 flows, Tier1 (свой AudioReact) vs Tier2 (audiobridge.js).

Что строим

Audio-reactive AV-инсталляция: аудиовход (системный звук или микрофон) → разложение на признаки (spectral / onset / envelope) → mapping в визуальные параметры → рендер в TD или в браузер. Latency <50 ms (иначе движение «отстаёт» от музыки на слух).

Получение аудио в TD

WASAPI loopback напрямую в текущей сборке TD недоступен (memory project_td_audioreact_audio). Решение — VB-Cable:

SYSTEM AUDIO ──► VB-Cable (virtual sink) ──► TD Audio Device In (loopback)
                            │
                            └─► Windows Listen-Through ──► реальные колонки

VB-Cable перехватывает системный звук, TD читает его как обычный микрофонный вход, а Listen-Through отдаёт звук на физические колонки/наушники. Без Listen-Through пользователь не слышит то, на что реагирует визуал — частая ошибка первого захода.

Архитектура контейнеров в .toe:

INPUT  (subcontainer)   audio in CHOP + analyzer CHOPs
OUTPUT (subcontainer)   visual rendering, projector mapping

Эталонный файл проекта — Clode_React.toe на Десктопе.

Признаки аудио

ПризнакЧто показываетГде применять
Volume RMSГромкостьБазовая «амплитуда» движения
Spectrum (FFT)Распределение по частотамЦвет, форма, текстура
OnsetРезкие изменения (барабан, удар)Триггеры, мигание
BeatТемпPulse, ритмический мотив
Envelope (ADSR)Атака/удержание/спадПлавное затухание

Простое правило: для удара/триггера — onset; для плавного движения — envelope; для цвета/палитры — spectrum.

Audio-analyzer

Из практики (memory project_td_audioreact_flows): отдельный Python-модуль audio_analyzer.py — Фаза B. Делает первичную обработку до того, как сигнал попадает в TD-flow.

import numpy as np
import librosa

def analyze(samples: np.ndarray, sr: int = 48000):
    rms = float(np.sqrt(np.mean(samples ** 2)))
    spec = np.abs(np.fft.rfft(samples)[:1024])
    onset = float(librosa.onset.onset_strength(y=samples, sr=sr).mean())
    centroid = float(librosa.feature.spectral_centroid(y=samples, sr=sr).mean())
    return {
        "rms": rms,
        "spectrum_bands": spec.tolist(),
        "onset": onset,
        "centroid": centroid,
    }

В TD передаём через DAT (для метаданных) или CHOP (для непрерывных значений). Опять же — не HTTP, latency-критично.

Tier 1 vs Tier 2

В коллекции 27 flows (memory project_td_audioreact_flows) есть два уровня:

  • Tier 1 — собственный AudioReact-стек, работает целиком в TD. Минимум зависимостей, латентность низкая, но писать «с нуля».
  • Tier 2 — через audiobridge.js, готовые мосты из web/Wekinator. Быстрее старт, но добавляется ~30 ms на bridge.

Tier 1 — для прода (фестиваль, клиент-инсталляция). Tier 2 — для прототипа или AV-вечера, где допустима небольшая задержка.

Также есть 55 factory presets — отправная точка для скетча, не финальный продукт.

Реальный кейс: AV-вечер av_jam_lab_demo

Из memory project_av_jam_lab_demo: серия AV-вечеров, первый демо 2026-05-08 в офис-лабе (24 часа), tdmkii + TD на 4070, инсталляции A/B/C/F.

Стек:

  • A — звук → spectrum → 3D-частицы;
  • B — onset → trigger generative video (StreamDiffusion из A9);
  • C — envelope + centroid → цвет проекции;
  • F — голосовой вход → text → визуальная типографика (через STT — см. A11).

Подготовка за день: загружаем 55 factory presets, выбираем ближайшие к концепции, настраиваем VB-Cable, тестируем 30 минут на живом звуке, фиксируем сетап. Без factory presets — день уходит только на компоновку базовых CHOPs.

Scene Player

Из memory project_td_audioreact_flows: Scene Player — Фаза C. Это управляющий слой над flows: переключение между сценами по timeline, MIDI, OSC или горячим клавишам.

Архитектура простая: один параметрический контейнер хранит активную scene, при переключении меняются routes к выходным TOPs.

TIMELINE / MIDI / OSC ──► Scene Player ──► scene_id (int)
                                              │
                                              ├──► [0] flow_a active
                                              ├──► [1] flow_b active
                                              ├──► [2] flow_c active
                                              └──► [3] flow_f active

Без Scene Player перформер прыгает между .toe-файлами вручную — это видно в проекции.

Подводные камни

  • WASAPI loopback недоступен в вашей сборке TD — не тратьте часы на поиск «как же его включить», ставьте VB-Cable.
  • Listen-Through забыли — публика не слышит, но видит реакцию. Заметьте сами на 30-секундном тесте перед публикой.
  • Smoothing подавляет onset — если onset CHOP сглажен, удары пропадают. Сглаживание — только на envelope, не на onset.
  • Большой буфер FFT — спектр «отстаёт». Держите 512–1024 семпла на блок FFT при 48kHz.
  • TD main-thread hang — известная проблема (memory project_td_audioreact_hang). Не запускайте WS-серверы в одном потоке с frame_exec.

Recap

  • VB-Cable для системного аудио в TD; Listen-Through — обязательно.
  • 6 базовых аудио-признаков; для триггеров — onset, для движения — envelope, для палитры — spectrum.
  • Tier 1 (свой AudioReact) — прод; Tier 2 (audiobridge.js) — прототип/AV-вечер.
  • Scene Player + Tier 1 flows — рабочая конфигурация для часовой инсталляции.
  • Latency-бюджет <50 ms; FFT 512–1024 на блок при 48kHz.
