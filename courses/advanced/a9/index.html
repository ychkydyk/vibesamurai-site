A9 · TouchDesigner + StreamDiffusion

Real-time generative video в TD: подача аудио и CV в StreamDiffusion, TdrDelay под Vulkan, latency-бюджет инсталляции. Реальные грабли поднятия патча.

Что строим

Real-time генеративная видео-инсталляция: TouchDesigner ловит аудио/CV/MIDI вход, кормит StreamDiffusion картинкой-условием, на выходе — стилизованный видеопоток в проектор или на экран с латентностью <200 мс.

Архитектура

[аудио/CV/MIDI] ─► TouchDesigner ─► tex_in ─► StreamDiffusion ─► tex_out ─► TD compositor ─► PROJ
                          │                       (LCM,           │
                          │                        SDXL Turbo,    │
                          ▼                        FLUX schnell)  │
                  control params ────────────────────────────────►│
                                                                  ▼
                                                              recording

Условие на вход StreamDiffusion — это не «приглушённое аудио», это картинка-condition: либо TD-рендер сцены, либо CV-вход с камеры, либо microspectrogram. Подача через shared GPU texture или через локальный named pipe / WS.

Бэкенды StreamDiffusion 2026

БэкендСкоростьКачествоКогда
LCM-LoRA на SDXL30–60 fps на 4070среднебазовый realtime

Latency budget инсталляции — обычно <200 ms суммарно (вход → выход на проектор), иначе аудитория ловит рассинхрон.

TdrDelay под Vulkan: критический шаг

Из практики (memory feedback_tdr_for_trt): перед запуском TRT-StreamDiffusion в TD обязательно установить TdrDelay = 60 в реестре Windows и перезагрузить машину. Без этого Vulkan падает с fatal на load_config.

PowerShell-сниппет:

# Запустить как Administrator
New-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\GraphicsDrivers" `
  -Name TdrDelay -Value 60 -PropertyType DWORD -Force
Restart-Computer

После reboot — TRT load_config не валится. Это обязательный шаг, не «может пригодится».

Типовой баг при поднятии TD + StreamDiffusion

Один из самых частых блокеров при первом запуске связки TD + StreamDiffusion на Windows — Python в TouchDesigner не видит StreamDiffusion-пакет, установленный в отдельный venv. Симптом: при импорте в TD выпадает No module 'StreamDiffusion' несмотря на корректную установку через pip. Фикс — через site.addsitedir() в init-скрипте TD-проекта.

Логика фикса:

# init.py в TD-проекте
import site, sys
from pathlib import Path

VENV = Path(r"C:\path\to\streamdiffusion-venv")
SITE_PKG = VENV / "Lib" / "site-packages"
site.addsitedir(str(SITE_PKG))

import StreamDiffusion  # теперь импортируется

TD по умолчанию запускает свой Python и не видит ваш venv. site.addsitedir() добавляет path к sys.path и регистрирует .pth-файлы. Просто sys.path.append — недостаточно.

После применения — Re-init контекста TD. Без Re-init старая sys.path сохраняется в памяти.

Подача параметров из TD в SD

Параметры (prompt strength, denoise, cfg, controlnet weight) меняются в реальном времени из TD-параметров. Передача — через shared dict или OSC:

# в StreamDiffusion runner
params = td_bridge.get_params()  # {"prompt_strength": 0.8, "cfg": 1.5, ...}
sd_pipeline.update(**params)
frame = sd_pipeline(condition_tex)
td_bridge.push_frame(frame)

td_bridge — тонкий named-pipe или Spout/Syphon канал, без HTTP. HTTP-вызовы между TD и SD добавляют 5–10 ms на запрос — недопустимо для real-time.

Latency-бюджет

┌─────────────────────────────────────────────┐
│  ВХОД (camera/audio/MIDI) →  TD             │   ~20 ms
│  TD compositor →  shared texture            │   ~15 ms
│  StreamDiffusion inference                  │   ~40-100 ms
│  Output texture →  TD                       │   ~15 ms
│  TD compositor →  projector                 │   ~30 ms
├─────────────────────────────────────────────┤
│  TOTAL                                      │   ~120-180 ms
└─────────────────────────────────────────────┘

Если бюджет вылетает (видно «лаг» в проекции относительно музыки) — режим деградации: понижаем размер condition tex до 512×512, переключаемся с FLUX schnell на SDXL Turbo, отключаем controlnet.

Подводные камни

  • «У меня 4090, всё взлетит» — на 8GB VRAM (4070) FLUX schnell идёт ~10 fps. На 4090 — лучше, но не «без бюджета». Размер модели и steps решают.
  • TD main-thread hang — известная проблема: TD-проект залипает на WebSocket-порте (типично :44444/:7878) даже без активного WS-клиента. Чек-лист: par_bindings / pack_json / frame_exec. Не диагностируйте на глаз.
  • Vulkan на драйверах NVIDIA <555 — нестабилен с StreamDiffusion. Обновляйтесь до последнего стабильного драйвера перед инсталляцией.
  • Тёплая модель в кэше GPU — первый кадр после старта 500 ms+, дальше — нормально. Прогрейте перед публичным запуском.
  • Перегрев при длинной инсталляции — 4 часа real-time inference на 4070 — это 70–80°C. Готовьте охлаждение или режим throttle.

Recap

  • Стек: TD (вход + compositor) + StreamDiffusion (inference) + shared texture/OSC (мост).
  • TdrDelay = 60 + reboot — обязательный шаг под TRT-StreamDiffusion.
  • Параметры передаём через named pipe / Spout / Syphon, не HTTP.
  • Latency-бюджет ~120–180 ms; режим деградации прописан заранее.
  • Прогрев модели до публики, контроль температуры.
