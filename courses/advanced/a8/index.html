A8 · Identity-preserving generation

Soul ID training, LoRA, character consistency через image-to-image и references. Когда что выбрать: артист, ведущий, повторяющийся персонаж серии.

Что строим

Пайплайн идентичности — серия артефактов (image + video), на которых одно и то же лицо/персонаж выглядит узнаваемо. Без identity-preservation серия из 12 постов превращается в «12 разных людей». С identity — это один узнаваемый бренд-персонаж.

Soul ID training

Из практики skill higgsfield-soul-id (сверка 2026-05-29):

  1. Сбор данных — минимум 15–20 фото человека: разные углы, освещение, выражения. Без очков, без шляп, без сильного бэкграунда. Все одного человека (не группа).
  2. Запуск тренировки — /higgsfield-soul-id с папкой фото → API возвращает reference_id.
  3. Стоимость и время — ~$3 per session, тренировка ~3–5 минут (на 15 фото). Это резко дешевле, чем выглядело в прежних оценках.
  4. Использование — в higgsfield-generate передаётся --soul-id <reference_id> для image и video.

Совместимые движки в higgsfield-стеке: text2image_soul_v2, soul_cinema_studio, Soul Cast, Soul Location.

LoRA: когда оправдан

LoRA подходит, если нужно зафиксировать стиль (например, фирменный иллюстрационный почерк) или character (вымышленный персонаж), и работать гибко:

  • регулярно дотренировывать (новые сцены, новые ракурсы);
  • использовать в open-weight стэке без vendor lock;
  • комбинировать с другими LoRA (стиль + персонаж).

Стоимость (сверка 2026-05-29 по FAL):

  • FLUX LoRA Fast training — ~$2 за полный run (linear по шагам).
  • FLUX.2 [dev] LoRA training — $0.008 за training step (9–50 фото, шагов ставят сотни–тысячи).
  • SDXL LoRA на Replicate — $3–10 в зависимости от длины training.

Размер веса: 50–150 MB. Совместимость: SDXL, FLUX dev, FLUX.2 [dev].

Reference image (one-shot)

Самый дешёвый вариант, когда identity нужна не «навсегда», а на 1–5 кадрах:

  • Nano Banana 2 принимает до 3 reference и держит лицо лучше FLUX;
  • FLUX Kontext — edit by reference, точечные правки одного исходника;
  • Higgsfield CLI с одним ref-файлом.

Минус: дрейфует на длинных сериях. Плюс: $0 на «подготовку», только стоимость генерации.

Реальный кейс: артистка Lubava

Из проекта project_lubava_platform (Любовь Соколова, handpan/вокал/перкуссия):

Задача — серия публикаций под React+Vite+TS платформу артистки: фото, превью клипов, motion-pin под СММ.

Подход: Soul ID training на 22 студийных фото артистки (разный свет, разные углы, нейтральные выражения). Стоимость: ~$3 per session, ~5 минут training. Результат: reference_id хранится в проекте, переиспользуется во всех генерациях.

Стек дальше:

  • Static: text2image_soul_v2 + бренд-токены платформы → серия портретов под обложки треков.
  • Motion: soul_cinema_studio (--soul-id <id> + cinematic prompt) → 8-секундные motion-pin под Instagram.
  • Live performance render: image-to-video с hero-кадром реальной съёмки + Soul ID для гарантии узнаваемости в стилизованных переходах.

Время серии из 8 артефактов: 1 рабочий день, ~$20–25 генеративки + $3 Soul ID (амортизируется на сотни кадров — fixed-cost практически растворяется в выходе).

Подводные камни

  • Soul ID на плохих исходниках — если фото одного освещения/ракурса, training даёт «один кадр в разных позах». Разнообразие исходников критично.
  • ip_detected — Higgsfield ловит лицо в reference ДО создания (см. reference_higgsfield_cli). Blur помогает частично; обходной путь — Soul ID на собственном лице/модели, а не на фото из интернета.
  • LoRA + цена кадра — для проектов с <100 кадрами LoRA редко окупается; reference image дешевле.
  • «Soul ID — навсегда» — нет. Через 6–12 месяцев модель-основа обновляется, Soul ID может требовать перетренировки. Бюджетируйте.
  • Лицо реального человека — нужно согласие. Юридически и этически. Договор с артистом / моделью / клиентом до начала тренинга.

Recap

  • Три подхода: Soul ID (реальное лицо), LoRA (стиль/character), reference image (1–5 кадров).
  • Soul ID — single payment ~$3, дальше копейки за кадр.
  • LoRA окупается на сотнях кадров и при необходимости гибкой дотренировки.
  • Reference image — для разовых задач; на длинной серии дрейфует.
  • Согласие лица — обязательно. Договор до тренинга.
