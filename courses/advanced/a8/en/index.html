A8 · Identity-preserving generation

Soul ID training, LoRA, character consistency via image-to-image and references. Which approach when: an artist, a host, a recurring brand character.

What we’re building

An identity pipeline — a series of artifacts (image + video) where the same face/character looks recognisable. Without identity preservation, a 12-post series turns into “12 different people.” With it, it’s one recognisable brand character.

Soul ID training

From the higgsfield-soul-id skill (verified 2026-05-29):

  1. Data collection — minimum 15–20 photos of one person: varied angles, lighting, expressions. No glasses, no hats, no strong background. All the same person (not a group).
  2. Training — /higgsfield-soul-id with a folder of photos → the API returns a reference_id.
  3. Cost and time — ~$3 per session, training ~3–5 minutes (on 15 photos). This is sharply cheaper than older estimates suggested.
  4. Usage — pass --soul-id <reference_id> to higgsfield-generate for image and video.

Compatible engines in the Higgsfield stack: text2image_soul_v2, soul_cinema_studio, Soul Cast, Soul Location.

LoRA: when it’s justified

LoRA fits when you need to pin a style (e.g. a signature illustration look) or a character (a fictional persona) and work flexibly:

  • regularly fine-tune more (new scenes, new angles);
  • use it in an open-weight stack with no vendor lock;
  • combine with other LoRAs (style + character).

Cost (verified 2026-05-29 on FAL):

  • FLUX LoRA Fast training — ~$2 per full run (linear in steps).
  • FLUX.2 [dev] LoRA training — $0.008 per training step (9–50 photos, hundreds–thousands of steps).
  • SDXL LoRA on Replicate — $3–10 depending on training length.

Weight size: 50–150 MB. Compatibility: SDXL, FLUX dev, FLUX.2 [dev].

Reference image (one-shot)

The cheapest option when identity is needed for 1–5 frames, not “forever”:

  • Nano Banana 2 accepts up to 3 references and holds the face better than FLUX;
  • FLUX Kontext — edit by reference, targeted edits to one source;
  • Higgsfield CLI with a single ref file.

Downside: drifts on long series. Upside: $0 setup, only the generation cost.

Real case: artist Lubava

From project_lubava_platform (Lyubov Sokolova — handpan/vocals/percussion):

The brief: a publication series for the artist’s React+Vite+TS platform — photos, clip previews, motion-pins for SMM.

Approach: Soul ID training on 22 studio photos of the artist (varied lighting, varied angles, neutral expressions). Cost: ~$3 per session, ~5 minutes training. Result: a reference_id lives in the project and is reused across every generation.

Stack from there:

  • Static: text2image_soul_v2 + the platform’s brand tokens → a series of portraits for track covers.
  • Motion: soul_cinema_studio (--soul-id <id> + cinematic prompt) → 8-second motion pins for Instagram.
  • Live performance render: image-to-video with a hero frame from a real shoot + Soul ID to guarantee recognisability across stylised transitions.

Time for an 8-artifact series: 1 working day, ~$20–25 of generative + $3 Soul ID (the fixed-cost dissolves across hundreds of frames).

Pitfalls

  • Soul ID on bad source material — if the photos all have the same lighting/angle, training produces “one frame in different poses.” Variety of source is critical.
  • ip_detected — Higgsfield catches a face in the reference before generation (see reference_higgsfield_cli). Blur helps partially; the safe path is Soul ID on your own face/model, not on photos from the internet.
  • LoRA + per-frame cost — for projects with <100 frames a LoRA rarely pays back; reference image is cheaper.
  • “Soul ID is forever” — no. Over 6–12 months the base model updates and Soul ID may need re-training. Budget for it.
  • A real person’s face — consent is required. Legally and ethically. A contract with artist / model / client before the training run.

Recap

  • Three approaches: Soul ID (real face), LoRA (style/character), reference image (1–5 frames).
  • Soul ID — single payment ~$3, then pennies per frame.
  • LoRA pays back across hundreds of frames and when flexible re-training is needed.
  • Reference image — for one-off tasks; drifts on long series.
  • Face consent is mandatory. Contract before training.
