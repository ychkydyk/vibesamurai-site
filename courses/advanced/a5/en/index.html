A5 · Image generation for a brand

FLUX 2, GPT Image 2, Nano Banana 2/Pro — which engine for which task. Brand-token prompt templates, tokenomics gate, a real agency case.

What we’re building

A production pipeline for image creative under a brand: hero for landing, an SMM series, article covers. The goal is repeatable quality in the brand’s look — not “a lucky frame once in ten attempts.”

Three engines in May 2026

Prices verified 2026-05-29 against the vendor docs.

EngineStrengthsWeaknessesCost
FLUX 2Photoreal, composition, the “editorial” lookWeaker text-in-image$0.03/MP (text2img), $0.045/MP (edit)
GPT Image 2Text in image, typography, schematics, designLess photoreal$0.005 low-q → $0.401 high-q (4K), by resolution
Nano Banana 2 / ProCharacter/reference, multi-ref consistencyWeaker photo-quality than FLUX$0.06 (0.5K) / $0.08 (1K) / $0.12 (2K) / $0.16 (4K)

For 1024×1024 (1 MP) the ballpark: FLUX 2 ≈ $0.03, Nano Banana 2 ≈ $0.08, GPT Image 2 ≈ $0.04 (medium-q). On batches the 2–3× gap really shapes the budget.

Selection rule:

                What do you need?
                       │
       ┌───────────────┼───────────────┐
       │               │               │
   PHOTOREAL       TEXT IN         REFERENCE /
   / EDITORIAL     IMAGE           CHARACTER
       │               │               │
    FLUX 2         GPT Image 2    Nano Banana 2

Don’t fight an engine’s weakness with prompt tricks — switch engines.

Prompt template on brand tokens

From A4: brand tokens carry an image_prompt_template. The final prompt is assembled from three layers:

[brand prefix] + [scene] + [negative suffix]
   ↑               ↑              ↑
   fixed           varies         fixed
   (≤30 words)     (≤50 words)    (≤20 words)

Example:

PREFIX:  high-contrast editorial, near-black background #0A0A0A,
         single chartreuse accent #C8FF00, Inter sans-serif typography,
         minimal composition, --ar 4:5
SCENE:   two figures from above, working at a desk, no faces visible,
         laptop closed, paper notes
NEGATIVE: neon glow, rainbow, vintage film grain, cute mascot

If the brand prefix drifts between posts, the series loses consistency. Pin the prefix as part of the design system, not the prompt body.

Tokenomics gate (mandatory)

Before any batch of image generations — estimate the cost and require an explicit confirmation from the author. This is rules feedback_tokenomics_confirm and feedback_analysis_tokenomics_gate.

def estimate(engine: str, count: int, mp_per_image: float = 1.0) -> dict:
    # Prices verified 2026-05-29.
    if engine == "flux-2":
        cost = 0.03 * mp_per_image * count            # $0.03 / MP
    elif engine == "flux-2-edit":
        cost = 0.045 * mp_per_image * count           # $0.045 / MP
    elif engine == "nano-banana-2":
        # 0.5K → $0.06, 1K → $0.08, 2K → $0.12, 4K → $0.16
        tier = {0.5: 0.06, 1.0: 0.08, 2.0: 0.12, 4.0: 0.16}
        cost = tier[mp_per_image] * count
    elif engine == "gpt-image-2":
        cost = 0.04 * count                            # medium-q 1024² ≈ $0.04
    else:
        raise ValueError(f"unknown engine: {engine}")
    return {"engine": engine, "count": count, "total_usd": cost,
            "approx_minutes": count * 8 / 60}

est = estimate("flux-2", 200, mp_per_image=1.0)
print(f"Batch {est['count']} frames via {est['engine']}: "
      f"${est['total_usd']:.2f}, ~{est['approx_minutes']:.1f} min.")
confirm = input("[y/N] continue? ").strip().lower()
if confirm != "y":
    raise SystemExit("cancelled")

200 FLUX 2 frames at 1 MP × $0.03 = $6. Without the gate, that leaves your wallet quietly; with the gate, it’s a deliberate decision.

Real case: 12-post series for an agency

Setup: 12 weekly posts under a brand (#0A0A0A + #C8FF00 + Inter). Each post = hero still + 6-second motion pin (motion in A7).

Stack:

  • brand tokens brand-tokens.json (see A4);
  • posts/q2.yaml — 12 entries (post_id, scene, mood);
  • runner — FAL Python SDK on top of FLUX 2;
  • cache — disk + JSON sidecars (see A2);
  • ffmpeg + Pillow — for logo and subtitle overlays.
import fal_client
from pathlib import Path

prefix = brand["image_prompt_template"]
negs   = ", ".join(brand["negatives"])

for post in posts:
    seed = stable_seed(post["id"])
    prompt = f"{prefix} — {post['scene']}"
    out, meta = run_step("image:hero", {
        "engine": "flux-2",
        "prompt": prompt,
        "negative": negs,
        "seed": seed,
        "ar": "4:5",
    }, runner=fal_flux_runner)
    Path(f"posts/{post['id']}_hero.png").write_bytes(out.read_bytes())

Result: one run produces 12 frames (1 MP each), ≈ $0.36 on FLUX 2, ~2 minutes. Any post can be re-generated with a new scene; the other 11 come from cache.

Pitfalls

  • No --ar in the prompt — the engine defaults to 1:1 while your feed is 4:5. Aspect ratio is part of the payload, not the scene.
  • Hex colours #0A0A0A in the prompt — FLUX 2 understands them but doesn’t always hit exactly. Pin via 1–2 reference images or a fixed per-brand seed.
  • Text on FLUX — don’t. Text elements belong on GPT Image 2 or as a Figma/Photoshop post-process.
  • Multi-ref in one call — Higgsfield CLI auto-coerces multi-ref into start_image (see memory reference_higgsfield_cli). Collapse the task to one ref or use Nano Banana 2 with an explicit reference field.
  • Auto-upload fails — Higgsfield CLI has auto-upload outages. Upload assets in advance and pass upload_id.

Recap

  • Three engines for three tasks: FLUX 2 (photoreal, editorial), GPT Image 2 (text, design), Nano Banana 2 (references, character).
  • Prompt = brand prefix (fixed) + scene (variable) + negative suffix (fixed).
  • Tokenomics gate before a batch is required, not optional.
  • Cache + lineage (from A2) make iterations cheap: change one scene, regenerate one frame.
