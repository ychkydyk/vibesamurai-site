A17 · Token economics across modalities

Cost per frame, per video second, per 1k tokens. Haiku → Sonnet → Opus cascade + prompt caching + Batch API. Mandatory gate before paid generation.

What we’re building

A single cost-accounting system across every studio modality: text (LLM), image, video, audio. The goal is to know up-front what a project costs, see the cost drivers, and require an explicit confirmation before any paid generation.

A 1 MP FLUX 2 frame costs roughly 1.3k of Opus output tokens. A 6-second Seedance clip — about 70 FLUX frames. This is why video is the most “accounting-heavy” modality.

Haiku → Sonnet → Opus cascade

From production (memory feedback_course_translation_pipeline): for batch tasks (like translating a course) use a cascade — Haiku → Sonnet → Opus + prompt caching + Batch API. Chat examples — Sonnet, final — Opus.

Logic (measured on output because Claude output = 5× input):

ALL REQUESTS ──► Haiku ($5/1M out)        ──► 80 % closed cheaply
                  │
       simple?    │
                  ├── yes ──► OUTPUT
                  │
                  └── no  ──► Sonnet ($15/1M out)  ──► 18 % of tasks
                              │
                              ├── acceptable? ──► OUTPUT
                              │
                              └── no ──► Opus ($25/1M out)  ──► 2 % of tasks
                                            │
                                            └──► OUTPUT

Effective output cost: 5 × 0.8 + 15 × 0.18 + 25 × 0.02 = 4.0 + 2.7 + 0.5 = $7.2 / 1M output. Routing everything through Opus = $25/1M, so the cascade saves ~3.5×. With Batch API (−50 %) and prompt caching (cached input −90 %) — up to 6–8× cheaper than the direct Opus path.

Prompt caching

On repeat calls with the same prefix Anthropic caches the prefix for 5 minutes. Cached input cost → 10 % of normal.

# first call: caches the prefix
client.messages.create(
    model="claude-sonnet-4-6",
    messages=[{"role": "user", "content": [
        {"type": "text", "text": LONG_SYSTEM_PROMPT, "cache_control": {"type": "ephemeral"}},
        {"type": "text", "text": USER_QUESTION},
    ]}],
)

# subsequent calls (within 5 min): prefix from cache, −90 % input cost

When it’s justified: batched requests with the same system prompt; an interactive agent with a long CLAUDE.md. Direct ROI: on a booth with 1000 questions/hour and an 8k-token system prompt, prompt caching saves tens of dollars per day.

Batch API

Anthropic’s Batch API: requests complete within 24 hours (usually much faster), at 50 % of normal price.

When justified:

  • non-interactive tasks (overnight course ingest, translation, archive classification);
  • batches of 1000+ requests;
  • OK to wait minutes-to-hours.

Inside the Haiku → Sonnet → Opus cascade, running through Batch API gives another −50 %. Combined with prompt caching — up to −80 % vs the bare interactive path.

Tokenomics gate (mandatory)

The main studio rule (memory feedback_tokenomics_confirm + feedback_analysis_tokenomics_gate): before any paid generation (LLM analysis, image, video) — compute the cost and wait for an explicit [y/N] confirmation.

No back doors, no bypass flags. Both in the admin and in the CLI.

def gate(label: str, cost_usd: float):
    print(f"\n=== TOKENOMICS GATE ===")
    print(f"Task: {label}")
    print(f"Estimate: ${cost_usd:.2f}")
    confirm = input("[y/N] continue? ").strip().lower()
    if confirm != "y":
        raise SystemExit("cancelled by user")

This isn’t “caution for the squeamish.” A forgotten loop over 200 videos = $480 in five minutes. The gate catches it before money moves.

Real case: an overnight course translation

Brief: translate Beginner + Middle (32 chapters × ~10k chars = 320k chars of source, ≈ 12k output tokens per chapter). Cascade:

  1. Haiku — first-pass translation for all 32 chapters (max_tokens=12000). Cost: 32 × 12000 / 1M × $5 (out) = $1.92.
  2. Sonnet — review of the Haiku output, flagging chapters that need polish. Cost: ~$1.8 (smaller output, ~3000 tokens per chapter).
  3. Opus — final pass on the flagged chapters (5 of 32). Cost: 5 × 12000 / 1M × $25 = $1.50.

Total: ~$5.2 without optimisations. Batch API (−50 %) + prompt caching (−90 % on cached input) bring it down to ~$1.5–2.

Head-on through Opus: 32 × 12k × $25 / 1M = $9.60. Not catastrophic at the scale of one course, but at the scale of 300 materials — tens-to-hundreds of dollars saved.

Pitfalls

  • Count input tokens too — output is more expensive, but input piles up on big prompts. Especially with RAG (large context).
  • Caching only works on an identical prefix — change one character in the system prompt and the cache misses.
  • Batch API isn’t interactive — answers arrive as a batch, not in real time.
  • “Local is free” — no. A 4070 doing 24/7 inference = +200–300 W = real money on electricity.
  • Hidden costs — Soul ID training, S3 storage for caches, Cloudflare bandwidth. Count them.

Recap

  • Prices shift every 1–3 months — re-verify before each project (table verification: 2026-05-29).
  • Claude output = 5× input; optimise around output.
  • A 1 MP FLUX frame ≈ 1.3k Opus output tokens; a 6-second Seedance ≈ 70 FLUX frames.
  • Cascade Haiku → Sonnet → Opus saves ~3.5× vs “everything through Opus”; + Batch API + prompt caching up to 6–8×.
  • Tokenomics gate before paid generation — mandatory. No bypass.
