A4 · Design system as a contract

Brand tokens (palette, typography, voice, taboo words) as a machine-readable contract for generative pipelines and Figma. Fewer briefs in words, more repeatability.

What we’re building

A machine-readable brand contract consumed simultaneously by: the agent’s code (Astro/React), Figma MCP, prompt templates for FLUX/Seedance, and motion presets for TouchDesigner. When brand tokens are JSON rather than a PDF guide, all of this works without manual re-syncing.

Brand contract

{
  "brand": "atlas-studio",
  "version": "2026-05",
  "palette": {
    "bg":      "#0A0A0A",
    "accent":  "#C8FF00",
    "text":    "#FAFAFA",
    "muted":   "#8A8A8A"
  },
  "typography": {
    "display": { "family": "Inter", "weights": [700, 800] },
    "body":    { "family": "Inter", "weights": [400, 500] },
    "mono":    { "family": "JetBrains Mono", "weights": [400, 500] }
  },
  "voice": {
    "tone":   ["calm", "concrete", "no pathos"],
    "tabu":   ["revolution", "unique", "innovative"],
    "person": "second-singular"
  },
  "image_prompt_template": "high-contrast editorial, near-black background #0A0A0A, single chartreuse accent #C8FF00, Inter-style sans-serif typography elements, minimal composition, --ar 4:5",
  "motion_prompt_template": "slow controlled motion, 24fps, low contrast camera move, no zoom-burst, no glitch",
  "negatives": ["neon glow", "rainbow", "vintage film grain", "cute mascot"]
}

This file is the single source of truth for the brand. Every other artifact is its derivative.

Where it plugs in

brand-tokens.json
   ├── site/tailwind.config        (via token transformer)
   ├── figma-tokens-plugin         (via Figma Variables import)
   ├── prompts/image/*.tpl         (jinja interpolation)
   ├── prompts/video/*.tpl
   ├── ts-types/brand.d.ts         (type-safe in TS code)
   └── td-presets/atlas-studio.toe (via TD parameters)

Change one file → brand changes everywhere, synchronously, without manual re-sync. That’s what “design system as code” actually means.

Case: creative agency in mid-rebrand

A typical scenario for design-system-as-code is a 5–8 person creative agency going through a rebrand. I’ll illustrate with a fictional brand “Atlas Studio” — a stand-in for an agency producing immersive campaigns for fashion / lifestyle / tech clients.

Old system — a Figma file with components plus a PDF guide. New system — brand-tokens.json in a git repo:

  • Brand tokens: #0A0A0A (bg) + #C8FF00 (accent) + Inter (display/body) + pill tags (border-radius 999px).
  • Source of truth: Figma Variables (via the Tokens Studio plugin) + brand-tokens.json in git, synced both ways.
  • Layout benchmark: a separate repo templates/kp-agency/ with ready scaffolds for landings, KPs, review sites. Skeletons are copied from there at the start of every new client project.
  • Generative prompt: “editorial monochrome composition, near-black background #0A0A0A, single chartreuse accent #C8FF00, geometric forms, no neon glow.”

When a new client project starts — copy brand-tokens.json, run it through the same templates, get a visually consistent set: site + KP + 12 image posts + 3 motion variants. If the client asks for “their” palette — override only palette and voice.tone; the other layers stay.

Similar approaches have been documented publicly by Pentagram (in talks on machine-readable identity), Universal Everything (in their material library), Field.io (in case studies for Mercedes-Benz). The principle is the same everywhere: a brand is code, not a PDF.

Prompt template with tokens

import json, jinja2
brand = json.loads(Path("brand-tokens.json").read_text())
tpl = jinja2.Template(brand["image_prompt_template"] + " — {{ scene }}")
prompt = tpl.render(scene="studio frame, two figures, no faces, minimalism")
# → "high-contrast editorial, near-black background #0A0A0A, single
#    chartreuse accent #C8FF00, Inter-style sans-serif typography
#    elements, minimal composition, --ar 4:5 — studio frame, two
#    figures, no faces, minimalism"

Every new prompt is built from template + scene. The template is versioned independently of the scene paragraphs.

Workflow with Figma MCP

Figma MCP lets the agent read and write the design file. As of May 2026 the official server ships 14 tools; the two you’ll use 90 % of the time are get_design_context and use_figma. Practical notes from production (see memory reference_figma_mcp):

  • use_figma writes into existing pages; new pages are NOT committed. Pre-create the draft page.
  • use_figma returns nothing — after a call you re-read via get_design_context to verify the commit landed.
  • resize breaks auto-layout hug — after resizing set layoutSizingHorizontal = "AUTO" again, otherwise the layout drifts.
  • Font style names — Figma stores them without internal spaces (Inter Bold, not Inter Bold-700).
  • Recommended setup — install the official plugin: claude plugin install figma@claude-plugins-official (Anthropic’s official marketplace, Feb 2026).

A figma-sync.mjs script reads brand-tokens.json, updates Figma Variables via MCP, and checks that the palette and typography match. Run it at the start of a project and on every rebrand.

Voice contract for the LLM

voice.tone and voice.tabu are passed into the LLM’s system prompt when it writes brand copy:

SYSTEM: You write copy for the brand {{brand}}. Tone: {{voice.tone | join(', ')}}.
Strictly forbidden words and turns of phrase: {{voice.tabu | join(', ')}}.
Person: {{voice.person}} (e.g. "you do", not "we do").

This isn’t advice to the agent, it’s a contract: if the model uses “revolution,” the copy is rejected and rewritten. Add a post-validator on top — a regex against the taboo list with exit code ≠ 0 on violation.

Pitfalls

  • A PDF guide is not a design system — while the brand lives as a human-readable PDF, the agent can’t “see” it. Only a machine-readable JSON/YAML counts.
  • Versioning — brand@2026-05 vs brand@2026-06. Old artifacts must remember the version (in the lineage from A2) so you can reproduce “exactly that one.”
  • Drift between tokens and Figma Variables — the sync script must fail CI when they diverge.
  • Over-detailed template — a long image_prompt_template suppresses the scene paragraph. Keep the template ≤30 words; everything else goes into scene.
  • Taboo list without a post-validator — the LLM will violate the rule regularly. Regex + reject are non-negotiable.

Recap

  • A brand contract is a JSON file with palette, typography, tone, taboo words, and prompt templates.
  • Plugs into code, into Figma (via MCP/Variables), into prompts, into TD presets — single source of truth.
  • Taboo list + post-validator (regex) — without them brand voice drifts.
  • Version the brand; artifact lineage has to remember the version.

Further reading

  • Design Tokens Community Group — Format spec — tr.designtokens.org/format. The token format compatible with Style Dictionary and Tokens Studio.
  • Style Dictionary — styledictionary.com. A translator from tokens into any format (CSS, iOS, Android, Figma).
  • Tokens Studio for Figma — tokens.studio. Plugin syncing Figma Variables with a token git repo.
  • Figma Variables docs — help.figma.com/hc/en-us/articles/15145852043927. The base model on which MCP sync sits.
  • Figma MCP server guide — help.figma.com/…/32132100833559. Current tool list and setup.
