A4 · Design system как контракт

Бренд-токены (палитра, шрифты, тон, табу-слова) как машинно-читаемый контракт для генеративных пайплайнов и Figma. Меньше брифов на словах — больше повторяемости.

Что строим

Машинно-читаемый бренд-контракт, который потребляют одновременно: код агента (Astro/React), Figma MCP, промпт-шаблоны для FLUX/Seedance, motion-presets для TouchDesigner. Если бренд-токены — JSON, а не PDF гайд, всё это работает без ручного перетыкания.

Контракт бренда

{
  "brand": "wedoit",
  "version": "2026-05",
  "palette": {
    "bg":      "#0A0A0A",
    "accent":  "#C8FF00",
    "text":    "#FAFAFA",
    "muted":   "#8A8A8A"
  },
  "typography": {
    "display": { "family": "Inter", "weights": [700, 800] },
    "body":    { "family": "Inter", "weights": [400, 500] },
    "mono":    { "family": "JetBrains Mono", "weights": [400, 500] }
  },
  "voice": {
    "tone":   ["спокойный", "конкретный", "без пафоса"],
    "tabu":   ["революция", "уникальный", "инновационный"],
    "person": "second-singular"
  },
  "image_prompt_template": "high-contrast editorial, near-black background #0A0A0A, single chartreuse accent #C8FF00, Inter-style sans-serif typography elements, minimal composition, --ar 4:5",
  "motion_prompt_template": "slow controlled motion, 24fps, low contrast camera move, no zoom-burst, no glitch",
  "negatives": ["neon glow", "rainbow", "vintage film grain", "cute mascot"]
}

Этот файл — единственный источник правды о бренде. Все остальные артефакты — его производные.

Куда подключается

brand-tokens.json
   ├── site/tailwind.config        (через token transformer)
   ├── figma-tokens-plugin         (через Figma Variables import)
   ├── prompts/image/*.tpl         (jinja interpolation)
   ├── prompts/video/*.tpl
   ├── ts-types/brand.d.ts         (type-safe в TS-коде)
   └── td-presets/wedoit.toe       (через TD parameters)

Меняется один файл — синхронно меняется бренд везде, без ручной пересинхронизации. Это и есть «design system as code».

Реальный кейс: WEDOIT (rebrand с VOLT)

Из проекта агентства WEDOIT (бывшее VOLT, см. memory feedback_wedoit_brand):

  • Бренд-токены: #0A0A0A (bg) + #C8FF00 (accent) + Inter (display/body) + pill-теги (border-radius 999px).
  • Источник правды: Figma MDM design-system + brand-tokens.json в репо.
  • Эталон вёрстки: C:\Claude\kp-agency\ — оттуда копируется при старте нового КП/landing/review-сайта.
  • Промпт для генеративки: «editorial monochrome composition, near-black background #0A0A0A, single chartreuse accent #C8FF00, geometric forms, no neon glow».

Когда стартует новый клиент-проект (rubber_kp, ai-festival, flag-review, svib-review) — копируется brand-tokens.json, прогоняется через одинаковые шаблоны, получается визуально консистентный набор артефактов: сайт + КП + 12 image-постов + 3 motion-вариации.

Промпт-шаблон с токенами

import json, jinja2
brand = json.loads(Path("brand-tokens.json").read_text())
tpl = jinja2.Template(brand["image_prompt_template"] + " — {{ scene }}")
prompt = tpl.render(scene="кадр студии, две фигуры, без лиц, минимализм")
# → "high-contrast editorial, near-black background #0A0A0A, single
#    chartreuse accent #C8FF00, Inter-style sans-serif typography
#    elements, minimal composition, --ar 4:5 — кадр студии, две
#    фигуры, без лиц, минимализм"

Любой новый промпт в проекте строится из шаблона + scene. Шаблон версионируется отдельно от scene-абзацев.

Воркфлоу с Figma MCP

Figma MCP позволяет агенту читать и писать дизайн-файл. Важные нюансы из практики (см. memory reference_figma_mcp):

  • use_figma пишет в существующие страницы, новые страницы НЕ коммитятся. Создавайте страницу-черновик заранее.
  • use_figma не возвращает значения — после вызова надо перечитать через get_design_context, чтобы убедиться, что коммит прошёл.
  • resize ломает auto-layout hug — после resize вернуть layoutSizingHorizontal = "AUTO", иначе плывёт.
  • Стили шрифтов — Figma хранит как «Montserrat» без пробелов в weight (Inter Bold, не Inter Bold-700).

Скрипт figma-sync.mjs читает brand-tokens.json, обновляет Figma Variables через MCP, проверяет, что палитра и шрифты совпадают. Запускается перед началом проекта и при rebrand.

Контракт voice (тон голоса) для LLM

voice.tone и voice.tabu передаём в системный промпт LLM, который пишет тексты под бренд:

SYSTEM: Ты пишешь тексты для бренда {{brand}}. Тон: {{voice.tone | join(', ')}}.
Строго запрещены слова и формулировки: {{voice.tabu | join(', ')}}.
Обращение: {{voice.person}} (например, «ты делаешь», не «вы делаете», не «мы»).

Это не «совет агенту», это контракт: если он использует «революция» — текст откатываем и переписываем. Дополнительно — пост-валидатор: regex на табу-слова, exit code != 0 при нарушении.

Подводные камни

  • Гайд в PDF — это не дизайн-система — пока бренд лежит человекочитаемым PDF, агент его не «видит». Только машинно-читаемый JSON/YAML.
  • Versioning — brand@2026-05 vs brand@2026-06. Старые артефакты должны помнить версию (в lineage из A2), чтобы можно было повторить ровно «как тогда».
  • Несоответствие токенов и Figma Variables — синхронизатор должен валить CI, если расходятся.
  • Чрезмерная детализация шаблона — длинный image_prompt_template подавляет scene-абзац. Держите шаблон ≤30 слов, остальное — в scene.
  • Tabu-список без пост-валидатора — LLM регулярно нарушит запрет. Регулярка + откат — обязательны.

Recap

  • Бренд-контракт = JSON-файл с палитрой, шрифтами, тоном, табу-словами и шаблонами промптов.
  • Подключается в код, в Figma (через MCP/Variables), в промпты, в TD-пресеты — единый источник правды.
  • Tabu-список + post-validator (regex) — без них бренд-голос плывёт.
  • Версионируйте бренд; lineage артефактов должна помнить версию.

Дальше: посмотреть и почитать

  • Design Tokens Community Group — Format spec — tr.designtokens.org/format. Стандарт оформления design tokens, совместимый со Style Dictionary, Tokens Studio.
  • Style Dictionary — styledictionary.com. Translator токенов в любые форматы (CSS, iOS, Android, Figma).
  • Tokens Studio for Figma — tokens.studio. Плагин, синхронизирующий Figma Variables с git-репо токенов.
  • Figma docs — Variables — help.figma.com/hc/en-us/articles/15145852043927. Базовая модель variables, на которую ложится MCP-синхронизация.
