A6 · Marketplace + product photoshoot

Marketplace product cards (Wildberries / Ozon style) and brand product photography: lifestyle, closeup, CGI, restyle. So the output passes moderation and actually sells.

What we’re building

Two artifact sets for commercial tasks:

  1. Marketplace cards — main photo (compliant with the platform’s rules), 5–7 secondaries, A+ content modules.
  2. Product photoshoot — brand product photography in modes: product_shot, lifestyle_scene, closeup_product_with_person, moodboard_pin, hero_banner, social_carousel, ad_creative_pack, virtual_model_tryout, conceptual_product, restyle.

The split: marketplace is tuned for moderation compliance (white background, no overlays, product colour matches reality); photoshoot is tuned for brand and virality.

Marketplace pipeline

INPUT                  PROCESSING                   OUTPUT
─────────────────────────────────────────────────────────────
raw product photo  →   background → white            main.jpg
SKU description    →   colour normalisation          main_white_bg.jpg
platform rules     →   aspect 1:1 (Ozon)             secondary_lifestyle.jpg
                       secondaries generation         secondary_closeup.jpg
                       A+ modules                     aplus_module_1..N.jpg
                       compliance check               report.json

Compliance check is mandatory before upload. Moderators reject for text/logo/price on main, wrong background, cropped product, additional products in frame. Ship the rules as prompt + post-validator.

Ozon baseline rules (verified 2026-05-29):

  • main 1:1 = 2000 × 2000 px, shoot the raw at 3000 × 3000 or above.
  • product centred with 10–15% white/neutral margin (no clipping).
  • no text overlays, logos, prices, banners — even in the corners.
  • sRGB colour profile (not Adobe RGB / ProPhoto); strip EXIF.
  • MozJPEG → < 5 MB without visible degradation.

Wildberries and Amazon are similar in spirit. Exact numbers change every quarter — re-read seller.wildberries.ru/help/cards and seller-edu.ozon.ru before every new batch.

Marketplace skill

The repo ships a skill higgsfield-marketplace-cards. It hides compliance references behind the CLI: input = product description and category, output = full card set.

/higgsfield-marketplace-cards \
    product="women's winter boots, black leather, wool" \
    category="footwear" \
    sku="BT-2026-W-39" \
    refs=./input/boots_studio.jpg

What matters: the skill picks platform-specific prompt templates (WB/Ozon style). Don’t write the prompts by hand — you lose compliance.

Real case: a card set for an e-com brand

Brief: 1 SKU, 7 artifacts (main + 4 secondaries + 2 A+).

Stack:

  • main + 1 secondary closeup — product_shot + closeup_product_with_person via Higgsfield;
  • 3 secondary lifestyle — lifestyle_scene (urban evening, at home, sports);
  • 2 A+ modules — GPT Image 2 (specs + comparison vs competitor).

Cost per SKU: ~$1.5–3 depending on engines and resolution (7 frames × $0.05–0.16 via Nano Banana 2 / Higgsfield + overhead). Time: 20–30 minutes including compliance check. Before generative AI, the same volume = a studio shoot for 1–2 days + retoucher, budget from $300.

Compliance post-validator

def check_main_ozon(image_path: Path) -> list[str]:
    img = cv2.imread(str(image_path))
    h, w = img.shape[:2]
    errors = []
    if not is_white_bg(img, threshold=240):
        errors.append("main: bg not white (avg < 240)")
    if has_text_overlay(img):
        errors.append("main: text / logo / price detected")
    if has_multiple_products(img):
        errors.append("main: multiple products in frame suspected")
    if abs(w - h) > 4:
        errors.append(f"main: not 1:1 ({w}×{h})")
    if w < 2000 or h < 2000:
        errors.append(f"main: smaller than 2000×2000 ({w}×{h})")
    if product_margin_ratio(img) < 0.10:
        errors.append("main: margin from product to edge < 10%")
    if image_path.stat().st_size > 5 * 1024 * 1024:
        errors.append("main: file > 5 MB (run through MozJPEG)")
    return errors

A local check is cheaper than a moderator rejection plus a rebuild. Plug it in before upload.

Pitfalls

  • “It looks pretty, it’ll land” — a main without a compliance check is usually rejected. Visual quality ≠ marketplace quality.
  • Product colour drifts by 10–15 % — critical for apparel and cosmetics. Add colour normalisation and a reference frame with a colour chart.
  • Virtual try-on on a low-budget model — looks unhinged: faces drift, proportions wrong. Use Soul ID or load a reference model.
  • A+ modules overloaded with info — the user looks for a second. One thesis per module, not five.
  • ip_detected from Higgsfield — triggers before generation; blur helps partially (see memory reference_higgsfield_cli). Use depersonalised reference photos.

Recap

  • Marketplace and photoshoot are different pipelines. The first one is about compliance, the second one is about brand.
  • Don’t author prompts by hand: the skills (higgsfield-marketplace-cards, higgsfield-product-photoshoot) hold the assembly rules.
  • 7 cards ≈ $1.5–3 and 30 minutes; vs a studio — two orders of magnitude cheaper.
  • A compliance post-validator is mandatory before upload.
